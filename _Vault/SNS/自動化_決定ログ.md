# SNS 自動化 — 決定ログ (2026-04-24)

> 親: [[MOC_SNS]]
> この決定は [[SNS/戦略_2026-04-24]] の "Stage 3 は製品アカのみ" 条項を**個人アカに拡張**する例外判断である。リスクを受諾した上で実行する。

## 決定

- **アカウント**: @takumi_global (個人)
- **範囲**: 完全自動化 (生成 → 投稿 → アーカイブまで人手を介さない)
- **言語**: 日本語スタート ([[SNS/戦略_2026-04-24]] のクロスオーバー表に従う)
- **頻度**: 1 日 3 本 (cron で 07:00 JST 実行)
- **クレジット**: X API pay-per-use に自動チャージ有効化 + 初期 $10

## 受諾したリスク (2026-04-24 時点)

1. **algo 的 suppression** — 完全自動パターンは X 側で shadowban される可能性。**週次 engagement 監視で検知**
2. **LLM ハルシネーション** — 事実でない数字・固有名詞を流す可能性。**guardrail + Vault 抜粋オンリー生成で対処** (使用モデル: OpenAI gpt-4o)
3. **DM / リプ / 引用 RT の死角** — 完全自動化するとここが完全に手付かずになる。**週 1 の手動リプタイム (日曜 20:00) を運用ルールで義務化**
4. **財産の "本人が書いた" 性** — 将来の書籍・講演で「俺が書いた」と言えなくなる可能性。**投稿ログを全部 `公開済/` にアーカイブ、後で編集履歴として回収可能にする**
5. **炎上 1 回で不可逆**  — **kill switch 即起動 + NG ワードリスト + @mention 禁止でリスク低減**

## 標準手順 (Standard Operating Procedure)

### 通常運用 (自動)
- 07:00 JST: Vercel Cron が `sns-run.mjs --live --count=3`
- 生成 → guardrail → 投稿 → 公開済/archive 追記 → sns-posts.json 更新
- ログは `_Vault/SNS/ログ/YYYY-MM-DD.log`

### 週次レビュー (日曜 20:00 — 人間)
- `_Vault/SNS/公開済/` の先週 7 日分を読む
- 各投稿の engagement を追記 (いいね/リプ/保存)
- tone が崩れていたら `_Vault/SNS/tone_sample.md` を更新
- DM/リプに **人間として** 返信 (ここを自動化しない)

### 異常時 (kill switch)
```bash
# 即時停止 (次の cron から何も出ない)
cd "/Users/takuminittono/Desktop/iPhone app/lecsy-video"
echo "SNS_AUTO_DISABLED=true" >> .env.local

# Vercel 運用時:
vercel env add SNS_AUTO_DISABLED production
# value に "true" を入れる → 次の cron で即停止
```

### 発動条件 (以下のいずれかで即 kill switch)
- 炎上 (引用 RT バースト / リプ急増 / block 急増)
- 事実誤認を自分または読者が発見
- 自分のメンタルが SNS で壊れそうな時 (最優先)
- engagement が 3 週連続で低下 (tone 劣化シグナル)
