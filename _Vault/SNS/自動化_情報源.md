# SNS 自動情報収集 — 情報源マップ

> 親: [[MOC_SNS]] / cron は [[SNS/自動化_タイムテーブル]]

## 設計原則

1. **一次情報だけ取る** — 二次ソース (他人のツイート等) は手で選ぶ
2. **日次スナップショット** — `_Vault/SNS/Daily/YYYY-MM-DD.md` に全部まとめる
3. **Vault 駆動** — sns-run.mjs が生成時に Daily ファイルを sourceNote 候補として読む
4. **pillar とマッピング** — 各情報源はどの pillar に流すか決めておく

## 情報源 (T0 = 最優先)

### T0. 自分の Lecsy 数字

| ソース | 何を | どこから | 頻度 | 流す先 |
|---|---|---|---|---|
| **Stripe** | MRR / 新規契約 / churn | Stripe API | 日次 06:00 JST | P1 週末大玉 / 月次KPI |
| **Supabase DB** | DAU / 新規サインアップ / セッション数 | Supabase client | 日次 06:00 JST | P1 数字単発 |
| **Deepgram** | 今日の STT 使用量 (分/コスト) | deepgram-balance-check Edge Function | 日次 06:00 JST | P3 技術数字 |
| **App Store Connect** | DL 数 / ★評価 / 新規レビュー | ASC API (or itunesconnect-ruby) | 日次 08:00 JST | P1 数字単発 |
| **Sentry** | エラー件数 / 新規 issue | Sentry API | 日次 06:00 JST | P3 失敗告白 |

### T1. 開発活動

| ソース | 何を | どこから | 頻度 | 流す先 |
|---|---|---|---|---|
| **GitHub (自 commits)** | 昨日の commit messages + 差分概要 | `gh api` or gql | 日次 06:00 JST | P1 / P3 |
| **Obsidian Vault diff** | 過去 24h で更新された MD ファイル | `git log --since='24 hours ago'` | 日次 06:00 JST | P2 学び系 |
| **Claude Code セッション log** | 今日何やったか (session summaries) | `~/.claude/projects/*/sessions/` | 任意 | P1 / P3 |

### T2. B2B 営業 (パイロット校)

| ソース | 何を | どこから | 頻度 | 流す先 |
|---|---|---|---|---|
| **Gmail (営業)** | FMCC/UF ELI/Santa Fe 新着メール | Gmail API (label filter) | 日次 09:00 JST | P2 スレッド (許可範囲内) |
| **営業進捗ノート** | `Deepgram/営業/*.md` の diff | git log | 日次 | P2 |

→ **個人情報・校名・担当者名は guardrail で二重ブロック必須**

### T3. 外部市場 (競合/業界)

| ソース | 何を | どこから | 頻度 | 流す先 |
|---|---|---|---|---|
| **Otter.ai / Notta / CLOVA Note** | 新機能リリース | それぞれの release note RSS | 週 1 月 | 直接投稿しない、参考のみ |
| **Apple press / WWDC** | AI/Translation 系発表 | RSS (developer.apple.com/news) | 日次 | 危機察知 (P3 で触れる) |
| **Deepgram blog** | モデル/価格/機能更新 | deepgram.com/blog RSS | 日次 | P3 |
| **Hacker News / r/IndieHackers** | SaaS pricing / edtech 系 | HN front page API | 日次 | 素材 memo のみ |

### T4. OPT / 法務

| ソース | 何を | どこから | 頻度 | 流す先 |
|---|---|---|---|---|
| **USCIS news** | OPT / STEM OPT / H-1B 規則変更 | USCIS RSS | 週 1 月 | P2 緊急性高 |
| **legal/MSA etc.** の Vault diff | 契約テンプレ更新 | git log | 日次 | P2 (抽象化した学び) |

## Daily スナップショット ファイル構造

`_Vault/SNS/Daily/2026-04-25.md`

```markdown
# Daily Snapshot — 2026-04-25

## 数字 (T0)
- MRR: $0 (まだ Stripe Live 前)
- Supabase DAU: 12 (前日比 +3)
- Deepgram 使用量: 47 分 / $0.14
- App Store DL: 2 / 累計 ★ 4.5 (3 reviews)
- Sentry: 0 new error

## 開発 (T1)
### GitHub commits (昨日)
- feat(mp3): ...
- fix(sns): ...
### Vault diff
- Deepgram/EXECUTION_PLAN.md: 12 lines changed
- SNS/tone_sample.md: rewritten

## 営業 (T2)
- FMCC: 新着メールなし
- UF ELI: 返信待ち 3 日目

## 市場 (T3)
- Otter: 新機能 pull 該当なし
- Deepgram blog: 新 nova-4 βアナウンス ★ (P3 投稿候補)
- Apple news: WWDC 2026 basic info

## OPT (T4)
- USCIS news: 変更なし

---

## 今日の投稿候補 (gpt-4o が自動でここから選ぶ)

### 朝 08:00 候補
[T0 数字 × P1] 昨日の Deepgram 47 分 / $0.14。90 分講義 x 30 人 = $4.2/日 の実費感。

### 昼 12:30 候補
[T3 × P3] Deepgram nova-4 β アナウンス。nova-3 からのレイテンシ改善が気になる。

### 夜 21:30 候補
[T1 × P2] Vault の EXECUTION_PLAN.md を大幅更新した。90日プランの見直し理由...
```

## 実装段階

### Phase 1 (今日-今週): 骨格
- `sns-collect.mjs` スケルトン (T0 + T1 だけ)
- Supabase + GitHub + Deepgram の 3 ソース
- Daily MD 生成
- sns-run.mjs が Daily MD も sourceNote 候補に含める

### Phase 2 (6/1 ローンチ後): B2B
- Stripe MRR
- App Store Connect
- Gmail 営業フィルタ (T2)

### Phase 3 (MRR 発生後): 市場
- RSS (Otter / Deepgram / Apple / USCIS)
- Hacker News
- 競合 price monitoring

## セキュリティ

すべて `.env.local` に API key、Vercel/GHA Secrets に同期。
Gmail は OAuth 必要 → PoC は手動 export でも OK (T2 は Phase 2)。
