# SNS 自動情報収集 — 情報源 v3 (AI 最新情報発信者)

> 親: [[MOC_SNS]] / cron は [[SNS/自動化_タイムテーブル]]

## 設計原則

1. **一次ソースから直接取る** — 二次記事は使わない
2. **RSS / GitHub atom / API が使える所を優先** — scraping は最後の手段
3. **日次 snapshot** — `_Vault/SNS/Daily/YYYY-MM-DD.md` に top 10 items を集約
4. **投稿で必ず引用** — post 本文に source URL 必須
5. **AI 関連キーワードで filter** — HN のような汎用 feed は keyword で絞る

## AI キーワード filter (HN 等の汎用 feed 用)

- 英語: `AI`, `GPT`, `Claude`, `Gemini`, `LLM`, `OpenAI`, `Anthropic`, `agent`, `fine-tun`, `benchmark`, `RAG`, `embedding`, `token`, `context window`, `Llama`, `Mistral`, `xAI`, `Grok`, `reasoning model`, `vision`
- 除外: `artificial intelligence` 単独 (ノイズ多), `AI generated` (画像), `machine learning course` (教育広告)

- 朝 08:00 (P1/P2): item 1 (一次ソース・大発表)

## 投稿時の source 扱い

- sns-generate.mjs に Daily.md の 1 item を context として渡す
- generator は item の title + URL + summary を sourceContent として使う
- guardrail の fact-check は summary 内容に対して行う (完全ではないが最低限の hallucination 防止)
- post 本文に必ず URL を含める (voice の核)

## セキュリティ / rate limit

- 全 RSS 取得は `User-Agent` 付与 (礼儀)
- HN/Reddit は `If-Modified-Since` 使う
- 取得失敗は Slack warning、pipeline は止めない (冗長性)
