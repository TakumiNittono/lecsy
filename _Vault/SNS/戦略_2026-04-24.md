# SNS = 財産 戦略 (2026-04-24)

> 出典: [[学び/Work in Public 戦略]] + [[学び/Justin Welsh LinkedIn 3段階システム]] を SNS 全体に拡張
> 親: [[MOC_SNS]]

## 前提: なぜ動画じゃなく "SNS全体" を財産にするのか

- 動画は 1 本ずつのヒット狙い。**積まない。**
- SNS = フォロワー + アーカイブ + DM = **Lecsy 後も残る distribution**
- 6/1 ローンチ後の B2B 2社目/就職/投資家/顧問業の入口すべてがここに集まる

## 財産の定義 (Lecsy を畳んでも残るもの)

1. **個人 X アカウント** (@nittonotakumi) — Lecsy 後の 2 社目の発射台
2. **投稿アーカイブ** — OPT 日本人 × 米B2B SaaS 立上げ一次ソース。将来の本・LP・講演の素材
3. **DM リスト** — "返信してくれる人" のリスト。followers数より重い
4. **検索一次ソース化** — "ESL × AI" / "Deepgram コスト" / "iOS on-device STT" で nittono = 最初の検索結果

## 配分: 個人 8 : 製品 2

- 製品アカ (@lecsyapp) は Launch 期の拡声器。畳んだら消える
- 個人アカが **財産の本体**。投稿は個人アカから、必要なときに製品アカへ拡散

## 3 段階の自動化

### Stage 1 — 手動 + Buffer (今週〜6/1)
- Buffer 無料枠 (3ch / 10件予約) で X + LinkedIn
- 金曜 17:00「公開デー」に翌週分 7 本を仕込む
- ソースは **Vault 既存ノート (300+)**。1 から書かない

### Stage 2 — Claude 下書き生成 (6/1〜9/1)
- Cron で毎朝 `sns-draft.mjs` が Vault 差分を拾う
- `_Vault/SNS/ドラフト/YYYY-MM-DD.md` に候補 3 本を投下
- 人間は **レビュー 5 分** で Buffer に貼る (書く負荷 → 選ぶ負荷)

### Stage 3 — 完全自動化 (MRR 発生後)
- Vercel Cron + X API (pay-per-use) で定時投稿
- ただし **個人アカは半手動を維持**。完全自動は製品アカのみ (人間味が資産性のコア)
