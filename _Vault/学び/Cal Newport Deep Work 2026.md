# Cal Newport Deep Work 2026

> 出典: [[NotebookLM_学習/⑮ 生産性 個人運用]] / Cal Newport (Georgetown CS 教授)

## 3つの核心ルール（2026版）

### 1. ハイブリッド・アテンション・モデル
**「リモートワークの日（または個人開発の日）」を「一切の会議・メール・チャットを排除した純粋なディープワーク専用日」**と宣言。

lecsy 的適用:
- **火曜・木曜を「lecsy デー」**として固定
- 通知完全 OFF、メール 1日2回しか見ない
- OPT 本業の「非同期対応可」時間を狙い撃ち

### 2. AI 活用の厳格なルール
**AI に思考や執筆、ロジック作成の核心を丸投げしてはいけない**。

→ 「ディープワークの認知負荷」を AI で軽減しようとすると、
  **市場価値のある希少なスキル（キャリア・キャピタル）を磨けなくなる**。

**使って良い AI（シャローワーク）**:
- コード生成の骨格
- 議事録作成
- メール下書き
- 定型的なドキュメント作成

**自ら引き受けるべき（ディープワーク）**:
- lecsy の核心設計
- 新機能のアーキテクチャ
- ユーザー問題の本質理解
- ピボット判断

### 3. 45分間の集中限界の遵守
**人間の集中力は最大でも 45分〜52分が限界**。

3時間の連続集中を自分に強いるのは**「欠陥」ではなく「生理的不可能」**。
**45分ごとに短い休憩**（10-15分）を挟むことが長期的な持続可能性の鍵。

## 実行タイマー: 45/15 ルーチン

```
06:00-06:45 Deep Work (45分)
06:45-07:00 Break (15分：散歩・水・ストレッチ)
07:00-07:45 Deep Work (45分)
07:45-08:00 Break (15分)
08:00-08:30 朝食・シャワー
08:30-17:30 OPT 本業
（昼休みに1セット入れる）
18:00-18:45 Deep Work (45分: lecsy)
18:45-19:00 Break
19:00-19:45 Deep Work (45分: lecsy)
```

→ 1日 **4セット × 45分 = 3時間の真のディープワーク** が現実的上限。
