# MRR $1k → $10k 戦術

> 出典: [[NotebookLM_学習/⑤ インディーSaaS]] / Arvid Kahl, Rob Walling, Indie Hackers

## 原則

$1,000 の壁を超えた後は:
- **「理想的な顧客（ICP）」への集中**
- **収益性の最適化**

が鍵。機能追加ではなく**顧客層の整理**フェーズ。

## 戦術 4つ

### 1. 高チャーン顧客の切り捨て
- 低価格プランで**手間のかかる顧客**は、チャーン率を高め成長を停滞させる
- 最低価格を引き上げ、**より深刻な悩みを持つ層**（より大きな語学学校など）にシフト
- lecsy 具体: 「$29/月」プランを廃止し、「$299/月」から開始

### 2. 「バリュー・メトリック」に基づいた価格改定
月額固定ではなく、**顧客が価値を感じる変数**に連動:
- lecsy 例: 生徒数 / 講師数 / 処理される願書数
- WP Engine は価格体系を $29 からに変更したことで爆発的成長

### 3. 穴の空いたバケツを塞ぐ
広告を増やす前に、**トライアル → 課金への転換率**を改善。

やるべきこと:
- オンボーディングフローの磨き込み（最初の15分で Aha 体験）
- メール自動化（Day 1, 3, 7, 14 の適切なナッジ）
- 失注理由のヒアリング（退会者全員に手動メール）

競合より高くても、**特定のインフラや機能**（lecsy なら自動文字起こし）で差別化できれば、顧客は納得して支払う。

### 4. 高頻度・高緊急性の課題解決
毎日直面する、**痛みの強い課題**を解決することで、
サービスを「**あれば便利**」から「**不可欠な存在**」に変える。

lecsy 例: 出欠は毎日 / 願書は毎週 → **毎日開かれる**機能にする

## 成功シグナル

- LTV/CAC が **3以上**
- Net Revenue Retention が **110%以上**（既存顧客が自然に増額）
- 解約率が **月5%以下**
- チャーンが減って **紹介が増える**

## lecsy の具体アクション

- [ ] 最低価格を $299/月に設定（小規模校は切る）
- [ ] 「生徒数ベース」のバリュー・メトリック導入
- [ ] オンボーディング Aha 体験を15分以内に設計
- [ ] 解約者全員に手動 Zoom インタビュー依頼
