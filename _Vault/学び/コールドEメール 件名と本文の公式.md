# コールドEメール 件名と本文の公式

> 出典: [[NotebookLM_学習/① 米国B2B営業]] / Patrick Dang

## 原則

**3〜4行以内**が最も読まれる。長いメールは開封されても読まれない。

## 件名の3公式

| 公式 | 例 |
|---|---|
| **理想の結果 (Desired Outcome)** | 「[学校名] の留学生登録数を 20% 増やす方法」 |
| **痛み (Pain)** | 「願書処理の Excel 管理に疲れていませんか？」 |
| **特定の参照 (Reference)** | 「UF ELI の最新の取り組みを拝見して」 |

## 本文の3-4行構造

```
[1行目] 関連する導入 — 相手へのリサーチに基づいた個人的な一言
[2行目] 痛みの提示 — 語学学校が直面する具体的な問題
[3行目] 解決策/好奇心 — 「lecsy は、このプロセスを自動化し、スタッフの事務時間を半分にします」
[CTA]   低摩擦な提案 — 「もしご興味があれば、こちらのカレンダーから [リンク]」
```

## テンプレ例（FMCC宛）

```
件名: UF ELI の新 ESL プログラム展開を拝見して

Ryoko 様、

Fulton-Montgomery CC の SUNY 統合プログラムで、
願書処理を Excel で手動管理されていると伺いました。
lecsy は留学生の願書・ビザ期限・出欠を自動で紐付けて、
週10時間の事務時間を削減します。

15分だけ中身をお見せできればと思うのですが、
こちらから都合の良い時間を選べますか？ [リンク]

Takumi Nittono
lecsy
```
