# 辞めない精神衛生 — Your whole goal is to not quit

> 出典: [[NotebookLM_学習/⑤ インディーSaaS]] / Courtland Allen (Indie Hackers)

## 前提

**個人開発における最大の失敗原因は、資金切れではなく**:
> **「ファウンダーの挫折（辞めること）」**

つまり辞めなければ、負けない。

## 技法 4つ

### 1. 「社会的責任」の活用
- 毎週の進捗を**ニュースレターや SNS で宣言**
- 他人の目があることで、モチベーションが低い日も **「恥をかきたくない」**という力が働く
- lecsy 実践: **毎週金曜に X / Substack に開発進捗を公開**

### 2. 結果ではなく「プロセス」を楽しむ
- MRR $10k という目標だけに執着すると、苦しい時期に耐えられない
- **日々の課題解決そのものをゲームのように楽しむ**マインドセットが必要
- lecsy 実践: 「今日も1つバグを倒した」を小さく祝う

### 3. 期待値を下げる
- 「一発で成功する」と思わず、「**失敗しても学習になる**」と期待値を下げる
- 精神的ダメージを最小限に抑えられる
- **幸福は低い期待値から生まれる**
- lecsy 実践: MRR $1k までは「**まだ学習期**」と認定。失敗を歓迎

### 4. 「孤独」を避ける
- Indie Hackers のようなコミュニティに参加
- 他の人が同じように苦労している姿（**マラソンの並走者**）を見ることで、
- 「自分だけではない」と勇気を得る
- lecsy 実践: Indie Hackers、Hacker News、日本の個人開発者 Discord にジョイン

## 危険シグナル（辞めたくなる兆候）

- [ ] 2週間、ユーザーと話していない
- [ ] 3日、コードを書いていない（休息ではなく逃避の場合）
- [ ] 比較対象が急に増える（「XX社はもう MRR $50k…」）
- [ ] 新しいアイデアに浮気し始める（現プロジェクトを深掘りせず）

→ 1つでも YES なら、**1時間だけパイロット校とメッセージやり取りする**。大抵戻ってくる。
