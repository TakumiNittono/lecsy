# Work in Public 戦略

> 出典: [[NotebookLM_学習/⑪ マーケティング・グロース]] / Nathan Barry (Kit / ConvertKit)

## 原則

製品開発のプロセスや裏側をドキュメンタリー形式で共有し、**顧客をジャーニーに巻き込む**。

## 3つの技法

### 1. 「公開で作る」
- リブランドの裏側
- 機能開発の意思決定プロセス
- 失敗とピボット

→ 単なる機能比較を超えた**信頼と感情的な繋がり**が生まれる。

### 2. 「95% 価値提供 + 5% 販売」
- コンテンツの **95% は無料の教育**
- 残り **5% でサービス/製品を紹介**

Demand Curve の Neal O'Grady も同じ比率を推奨。

### 3. 「顧客をヒーローにする」
- 自社製品ではなく、**顧客がどのように成功しているか**をケーススタディ
- 顧客に**「語るべき言葉（Talking Points）」**を与えることで口コミ誘発

### チャネル配分
- **LinkedIn**: B2B ターゲット向けの学び（週1本）
- **X/Twitter**: 技術・個人開発者向け（週3-5本）
- **Substack**: 長文デープ解説（隔週）
- **YouTube**: 月1本の深掘り（10分程度）
