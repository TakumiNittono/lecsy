# Design Sprint — ソロ開発者版

> 出典: [[NotebookLM_学習/⑭ UX UI デザイン]] / Jake Knapp (GV → AJ&Smart)

## 原則

Google Ventures で開発された**5日間の意思決定メソッド**を、
**ソロ開発者の週次サイクル**に圧縮。

## 1週間の型

### 月曜日: Map
- **ユーザーが「Aha体験」に至るまでのステップ**を可視化
- 最もリスクの高い箇所（例: 最初の生徒登録）に焦点
- **「今週はこの1箇所だけに集中する」**と決める

### 火曜日: Sketch
- **競合アプリから良いアイデアを盗む**（Duolingo, Notion, Google Classroom等）
- 自分のアイデアを紙に書き出す（**8つのアイデア × 8分**）
- 最良の1つを選ぶ

### 水曜日: Decide
- 複数案から、**最も「学習効果が高い」**ものを選ぶ
- **絵コンテ（Storyboard）**を作成: 「ユーザーが画面Aで何を見て、何を押すか」

### 木曜日: Prototype
- **本物のアプリを作るのではなく、Keynote / Figma でモックアップ**
- ユーザーが**「本物だ」と信じられる外見**
- 8時間以内で完成させる

### 金曜日: Test
- **5人**のターゲットユーザーに見せる
- **5人いれば課題の85%が発見できる**
- 操作を観察しながら、思考を声に出してもらう（Think-Aloud）

## ソロで回すコツ

### 「強制力」の活用
**「金曜日に実際のユーザーにテストしてもらう」**という予定を月曜に確定。
この**締め切りが生産性を生む**。

### 5人のテスターの見つけ方（lecsy 例）
- パイロット校の先生（FMCC, UF ELI）2名
- 過去にデモした見込み顧客 1名
- ESL 教師 FB グループから応募 1名
- 友人の留学生 1名

### Zoom でテスト
- **画面共有**してもらいながら、新機能を触ってもらう
- 「今、何を考えていますか？」を繰り返し聞く
- 録画して後で詳細分析

## lecsy 向け Sprint ネタ候補

- **生徒オンボーディングフロー**（最初の5分で Aha 体験）
- **録音 → 保護者通知の2タップ完結**
- **教師ダッシュボード（週次サマリー）**
- **願書の自動取り込み**
