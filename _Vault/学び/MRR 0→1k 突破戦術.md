# MRR $0 → $1k 突破戦術

> 出典: [[NotebookLM_学習/⑤ インディーSaaS]] / Starter Story, Indie Hackers, Arvid Kahl

## 原則

最初の $1,000 を稼ぐには、**製品開発よりも**:
- 「適切な課題の特定」
- 「配信チャネルの確保」

に全力を注ぐ。

## 戦術 4つ

### 1. オーディエンス・ファーストの徹底
製品を作る前に、**ターゲットがどこに集まり、何に不満を漏らしているか**を徹底リサーチ。
- 語学学校の場合: TESOL 系 Facebook、ESL 教師の Reddit、ELIA フォーラム
- 毎日1時間、書き込みを読み込む

### 2. 配信チャネルからの逆算
「良いものを作れば人は来る」は**誤解**。
最初から「**どう届けるか**」を計画する。

- 既存のコミュニティを活用
- その文脈に合った形で価値を提供（売り込み一切なし）

### 3. SLC (Simple, Lovable, Complete) の構築
**機能を削ぎ落とした壊れた MVP ではなく**、
小さくても「シンプルで、愛され、完結している」製品を目指す。

- lecsy の SLC 例: 「講師が出欠を2タップで送信 → 月末に保護者宛レポート自動送信」のみ
- 壊れた MVP: 全部できるけどどれも半端

### 4. 初期の泥臭い検証
- **100通以上**の個別メールを送る
- インタビューで課題を深掘り
- Pat Walls は Reddit でケーススタディを共有して初期トラフィック獲得

## lecsy の具体プラン

### Week 1-2: オーディエンス確認
- TESOL / ESL 教師 FB グループ 10個 ジョイン
- 毎日1時間、投稿を読んで痛みをノート化（[[_Vault/研究/ESL_痛み]].md 作成）

### Week 3-4: SLC 定義
- 痛み Top 3 から1つを選択
- 2週間で**動くプロトタイプ**（1機能のみ）

### Week 5-8: 泥臭い獲得
- FB / Reddit に誠実な投稿（ユーザーインタビュー募集）
- 100人にコールドメール
- 10人と Zoom、3人に有料使用してもらう

## 成功シグナル

- MRR $1k 到達前に最低**10人が毎日使う**状態
- 紹介が自然発生（「あの学校も使ってるよ」）
