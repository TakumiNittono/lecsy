# Claude プロンプト設計5原則

> 出典: [[NotebookLM_学習/⑨ AI LLM アプリ開発]] / Anthropic 公式

## 5原則

### 1. 構造化された構成
プロンプトは以下の順序で構成:
```
① 役割の定義
② タスクの説明
③ コンテキスト（動的データ）
④ 詳細な指示
⑤ 例（Few-shot）
⑥ 出力形式
```

### 2. XML タグの活用
`<context>` や `<instructions>` のようなXMLタグで囲むことで、Claudeは**情報の境界を明確に理解**。

```xml
<context>
{学生データ}
</context>

<instructions>
上記データから要約を作成
</instructions>
```

### 3. 思考の順序を指定
複雑なタスクでは、**「まず何を確認し、次に何を判断すべきか」**を論理的なステップで指示。

### 4. 出力形式の厳格化（Pre-fill）
JSON や特定 XML タグでの回答を求める際は、**回答の冒頭部分を `{` や `<verdict>` で始めるように**指定すると解析エラーを防げる。

### 5. 自信のない回答の抑制
> 「確信が持てない場合は、推測せずにその旨を伝えること」

という指示でハルシネーション防止。

## lecsy での適用

- **SummaryService.swift** のプロンプトに XML タグ導入
- 要約出力は `<summary>` で pre-fill
- 「ソースに無い情報は書かない」指示を徹底
