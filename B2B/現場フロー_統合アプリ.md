# Lecsy B2B 現場フロー（B2C/B2B統合アプリ版）

> 最終更新: 2026-04-05
> 方針変更: B2CとB2Bを別アプリにせず、**1つのアプリで統合運用**する

---

## 0. 方針変更の背景

### 従来の計画（撤回）
- lecsy（B2C）と lecsy-business（B2B）を別アプリとしてApp Storeに2つ出す
- B2Bは英語専門、B2Cは多言語対応

### 新方針（確定）
- **1つのアプリ「Lecsy」で B2C / B2B の両方をカバーする**
- ユーザーは個人としても使え、組織に所属すれば組織機能が解放される
- App Storeの審査・メンテナンスが1本で済む
- 個人ユーザーが組織に招待されたとき、アプリの再インストールが不要

### 統合の利点
1. **営業現場**: デモ中に「生徒が普段使ってるアプリに組織機能が追加されるだけ」と説明できる
2. **導入障壁の低さ**: 既にLecsyを個人で使っている生徒はそのまま組織に参加できる
3. **バイラル効果**: B2C → B2B への自然な導線（個人利用 → 学校が一括契約）
4. **開発効率**: コードベースが1つ、バグ修正も1回で済む

## 1. 全体フロー概要

```
【営業フェーズ】
  営業訪問 → デモ → パイロット提案 → パイロット開始
    ↓
【導入フェーズ】
  組織作成（Web） → 先生をメンバー追加 → 先生が生徒を追加
    ↓
【教室フェーズ】
  生徒がApp Storeからインストール → 組織に参加 → 授業中に録音・文字起こし
    ↓
【運用フェーズ】
  先生: Web管理画面で利用状況確認 / AI機能（Glossary, Cross-Summary）
  管理者: 利用統計・請求管理
    ↓
【契約フェーズ】
  パイロット結果報告 → 有料プラン提案 → Stripe課金開始
```

## 2. 営業訪問〜デモの流れ（現場15分）

### 2.1 訪問前の準備

| 準備項目 | 詳細 |
|---------|------|
| iPhone充電 | 100%にしておく |
| Lecsyの動作確認 | テスト録音→文字起こし→AI要約が正常に動くことを確認 |
| デモ用組織 | `demo-school` スラッグで事前に作成しておく |
| オフライン確認 | 機内モードでも録音・文字起こしが動作することを確認 |
| 名刺 | 持参 |
| 1ページャー | 印刷して持参（Canvaで作成） |

### 2.2 デモの実演（15分構成）

#### 0:00〜3:00 — ヒアリング（売り込まない）

```
「デモの前に2つ質問させてください:



 2. 今、リスニング練習にどんなツールを使っていますか？」
```

**ポイント**: 相手の課題を聞くことで、デモの見せ方をカスタマイズする。

#### 3:00〜8:00 — ライブデモ

```
[iPhoneでLecsyを開く]

1. 録音ボタンを押す
2. 英語で30秒話す → リアルタイムで文字起こしが画面に表示される
3. 「ご覧ください。全てこのiPhoneの中で処理されています。
    WiFiを切った状態です。クラウドには一切送信されません。」
4. 録音を停止 → AI要約を生成して見せる
5. 「授業後に生徒がこの画面を開くと、先生が何を言ったか
    一字一句確認できます。」
```

**統合アプリならではのポイント**:
```
「このアプリ、生徒が個人で無料ダウンロードして使えます。
 学校が契約すると、管理画面から生徒の利用状況が見えるようになり、
 AI機能（用語集自動生成・クロスサマリー）が組織全体で使えます。
 生徒はアプリを入れ替える必要はありません。
 今使ってるLecsyがそのまま、学校の機能付きにアップグレードされます。」
```

#### 8:00〜11:00 — 組織機能のデモ（Web管理画面）

```
[PCまたはiPadでWeb管理画面を開く]

1. demo-school の組織ダッシュボードを見せる
2. メンバー一覧 → 「先生・生徒がここに表示されます」
3. 利用統計 → 「誰が何回録音したか、日別・言語別で見えます」
4. AI機能 →
   - Glossary: 「録音から自動で用語集を生成します。
     学校固有の専門用語リストが蓄積されていきます」
   - Cross-Summary: 「英語の録音をスペイン語で要約、
     生徒の母語で理解を補助します」
```

#### 11:00〜15:00 — 提案とクロージング

```
「提案があります。
 まず先生2〜3人に無料で使っていただけませんか？
 費用は一切かかりません。4〜8週間使っていただいて、
 生徒の反応を見てください。

 導入はとても簡単です:
 1. 先生のメールアドレスを教えてください
 2. 私がWeb管理画面からメンバーとして追加します
 3. 先生がLecsyにログインすると、自動で組織に参加されます
 4. 先生が生徒にApp Storeのリンクを共有するだけ

 来週、先生にセットアップをお手伝いに来られますが、
 いつがご都合いいですか？」
```

## 3. 組織の作成と初期セットアップ

### 3.1 組織作成（Lecsy管理者が実施）

**実施者**: Lecsy側（Takumi）= スーパー管理者
**場所**: Web管理画面（`/admin`）

```
手順:
1. /admin ページにアクセス
2. 「組織を作成」をクリック
3. 以下を入力:
   - 組織名: 例「OHLA Orlando」
   - スラッグ: 例「ohla-orlando」（URLに使われる）
   - タイプ: language_school / university_iep / college / corporate
   - プラン: starter / growth / enterprise
   - 最大席数: 例 50
4. 作成ボタンを押す
```

**作成後に生成されるもの**:
- 組織ダッシュボード: `/org/ohla-orlando`
- メンバー管理: `/org/ohla-orlando/members`
- 利用統計: `/org/ohla-orlando/usage`
- AI機能: `/org/ohla-orlando/ai`

### 3.2 オーナーの設定

```
手順:
1. 学校のディレクター/オーナーのメールアドレスを聞く
2. Web管理画面 → メンバー管理 → 「メンバーを追加」
3. メールアドレスを入力、ロール「owner」を選択
4. 追加ボタンを押す → ステータス「pending」で作成される
5. 学校のオーナーがLecsyにログイン（Google認証 or メール認証）
6. ログイン時に自動でメールアドレスが照合され、pending → active に変わる
7. オーナーとして組織ダッシュボードにアクセス可能になる
```

## 4. メンバー追加の流れ（先生→生徒）

### 4.1 先生の追加

**実施者**: 組織のオーナー or 管理者
**場所**: Web管理画面（`/org/[slug]/members`）

```
方法A: 個別追加
  1. メンバー管理ページ → 「メンバーを追加」
  2. 先生のメールアドレスを入力
  3. ロール「teacher」を選択
  4. 追加 → pending状態で作成

方法B: 一括追加（CSVアップロード）
  1. メンバー管理ページ → 「一括招待」
  2. CSVファイルをアップロード（メール, ロール の2カラム）
  3. 最大100名まで一度に追加可能
  4. 全員がpending状態で作成される
```

**先生側の体験**:
```
1. 「Lecsyの組織アカウントに追加されました」（メール通知 ※今後実装）
2. App StoreからLecsyをダウンロード（既にあればそのまま）
3. Lecsyを開いてログイン（Google or メール）
4. ログイン時にメールアドレスが照合 → 自動で組織に参加
5. アプリ内に組織メニューが表示される
   - 組織の録音一覧
   - AI機能（Glossary, Cross-Summary）
   - クラス管理（生徒をクラスに割り当て）
```

### 4.2 生徒の追加

**実施者**: 先生 or 管理者
**方法は3つ**:

#### 方法A: メールアドレスで直接追加（管理画面から）
```
1. Web管理画面 → メンバー管理 → 「メンバーを追加」
2. 生徒のメールアドレスを入力
3. ロール「student」を選択
4. 追加 → pending状態
5. 生徒がLecsyにログインすると自動参加
```

#### 方法B: 招待リンク（先生がクラスで共有）
```
1. Web管理画面 → 「招待リンクを作成」
2. ロール「student」、有効期限を設定
3. 生成されたリンクを生徒にシェア（授業中にプロジェクターで表示、
   LMSに貼る、メールで送る等）
4. 生徒がリンクを開いてLecsyにログイン → 自動で組織に参加
```

#### 方法C: 一括CSV追加
```
1. 学校の事務局から生徒リスト（メールアドレス一覧）をもらう
2. CSVに整形（email, role=student）
3. 一括アップロード
4. 全員がpending → ログイン時に自動activate
```

### 4.3 メンバーの状態遷移

```
[追加] → pending（メールのみ、user_id なし）
          ↓ 本人がLecsyにログイン
        active（user_id が紐付く、全機能利用可能）
          ↓ 管理者が削除 or 学期終了
        removed（組織から外れる。個人アカウントは残る）
```

**統合アプリの重要ポイント**:
- 組織から外れても、個人としてLecsyは使い続けられる
- 個人の録音データは組織を離れても消えない
- 組織内で共有した録音のみ、アクセス権が失われる

## 5. 教室での利用フロー（生徒の1日）

### 5.1 授業前

```
1. 生徒がiPhoneでLecsyを開く
2. 組織に所属していれば、ホーム画面に組織名が表示される
3. 録音画面で言語を選択（英語がデフォルト、多言語対応）
```

### 5.2 授業中（録音・文字起こし）

```
1. 先生が話し始める
2. 生徒が録音ボタンをタップ
3. WhisperKit（オンデバイスAI）がリアルタイムで文字起こし
   - WiFi不要、完全オフラインで動作
   - バックグラウンドでも録音継続
   - 一時停止 / 再開が可能
4. 授業終了 → 停止ボタンをタップ
5. タイトルを入力（任意）→ 保存
```

**録音スペック**:
- 最大100分の連続録音
- チャンク処理: 3分未満は一括、3分以上は2分チャンク+3秒オーバーラップ
- タイムスタンプ付きセグメント生成
- クラッシュリカバリー対応（録音中にアプリが落ちても復旧可能）

### 5.3 授業後（復習・同期）

```
1. ライブラリで録音を選択
2. テキストを読みながら音声を再生
   - タイムスタンプ同期: テキストをタップするとその箇所から再生
   - 聞き取れなかった部分をテキストで確認
3. AI要約を生成（Pro機能 or 組織のプラン内）
   - 要約 / キーポイント / セクション分け
4. WiFi接続時にSupabaseへ自動同期
   - Web管理画面から先生も閲覧可能になる
```

### 5.4 組織AI機能の利用

#### Glossary（用語集自動生成）
```
対象: teacher以上のロール
1. Web管理画面 → AI機能 → 「用語集生成」
2. 対象の録音を選択
3. ターゲット言語を選択（日本語、スペイン語等）
4. GPT-4 Turboが10〜25の用語を抽出
   - 用語 / 定義 / 例文 / 難易度レベル
5. 組織の用語集データベースに自動保存（重複は上書き）
6. 生徒がアプリまたはWebから組織用語集を閲覧可能
```

#### Cross-Summary（クロスサマリー）
```
対象: student以上のロール
1. 英語の録音を選択
2. 出力言語を選択（例: スペイン語）
3. GPT-4 Turboが多言語要約を生成
   - 要約文（ターゲット言語で）
   - キーポイント
   - バイリンガル用語リスト
4. 生徒の母語で授業内容を理解補助
```

**AI利用制限（プラン別）**:

| プラン | 日次上限 | 月次目安 |
|--------|---------|---------|
| Starter | 10回/日 | ~300回 |
| Growth | 50回/日 | ~1,500回 |
| Enterprise | 200回/日 | ~6,000回 |

## 6. 先生の管理フロー

### 6.1 日常業務（Web管理画面）

```
/org/[slug]/ にアクセス

■ ダッシュボード
  - 今日の録音数
  - アクティブ生徒数
  - 最近の録音一覧

■ メンバー管理（/org/[slug]/members）
  - 生徒の追加 / 削除
  - ロール変更（student ↔ teacher）
  - 招待リンクの発行

■ クラス管理（/org/[slug]/classes）
  - クラスの作成（名前、言語、学期）
  - 生徒をクラスに割り当て
  - クラス別の利用統計

■ 利用統計（/org/[slug]/usage）
  - 日付範囲を指定して閲覧
  - メンバー別の録音回数
  - 日別推移グラフ
  - 言語別の利用割合

■ AI機能（/org/[slug]/ai）
  - Glossary生成・閲覧
  - Cross-Summary生成
  - 組織用語集の管理（カテゴリ、言語、検索）
```

### 6.2 学期の切り替え

```
学期終了時:
1. 卒業する生徒を一括削除（CSV or 手動）
   → 生徒の個人アカウントは残る（B2Cとして継続利用可能）
2. 新学期の生徒を一括追加（CSV）
3. 新しいクラスを作成
4. 組織用語集は蓄積されていく（リセット不要）
```

## 9. 統合アプリのUI設計方針

### 9.1 個人ユーザー（組織未所属）の画面

```
[ホーム]        → 録音画面（多言語選択可能）
[ライブラリ]    → 自分の録音一覧
[設定]          → アカウント、Pro課金、言語設定
```

- 組織機能は一切表示されない
- 通常のB2Cアプリとして完結

### 9.2 組織所属ユーザーの画面

```
[ホーム]        → 録音画面（組織名が上部に表示）
[ライブラリ]    → 自分の録音 + 組織で共有された録音
[組織]          → 組織メニュー（ロールに応じた機能）
  ├── 用語集（Glossary）
  ├── クロスサマリー
  ├── クラス一覧（teacher以上）
  └── メンバー管理（admin以上）
[設定]          → アカウント、組織切り替え（複数所属時）
```

### 9.3 組織参加時のアプリ内フロー

```
1. ユーザーがLecsyにログイン
2. バックエンドが organization_members テーブルを照合
3. pending のレコードが見つかる（メールアドレス一致）
4. 自動で active に変更、user_id を紐付け
5. アプリに組織メニューが出現
6. 「[組織名] に参加しました」のトースト通知
```

**アプリの入れ替え不要。ログインするだけで組織機能が追加される。**

## 10. データの分離とプライバシー

### 10.1 録音データの可視性

| 録音の可視性 | 説明 |
|-------------|------|
| `private` | 本人のみ閲覧可能（デフォルト） |
| `shared` | 同じクラスのメンバーが閲覧可能 |
| `org_wide` | 組織全体が閲覧可能 |

- 生徒はデフォルトで `private`。先生の判断で `shared` に変更可能
- 先生の録音（授業の録音）は `org_wide` がデフォルト

### 10.2 組織間のデータ隔離

```
- RLS（Row-Level Security）により、組織間のデータアクセスは完全に遮断
- 組織Aのメンバーは組織Bのデータを一切見られない
- スーパー管理者のみ全データにアクセス可能（運用・サポート用）
```

### 10.3 組織離脱時のデータ

```
生徒が組織を離れた場合:
  - 個人の録音データ（private）は本人のアカウントに残る
  - 組織と共有した録音は、組織側に残る（生徒からはアクセス不可に）
  - AI要約・用語集データは組織に帰属
  - 個人アカウントはB2Cとして引き続き利用可能
```

## 11. 営業〜運用の時系列フロー（1校の完全なライフサイクル）

### Week 0: 初回コンタクト
```
- 直接訪問 or メール or 電話
- 15分デモの実施
- パイロット（無料）を提案
- 先生2〜3人のメールアドレスを取得
```

### Week 0〜1: セットアップ
```
- Web管理画面から組織を作成
- ディレクターをownerとして追加
- 先生をteacherとして追加
- 先生のiPhoneでLecsyのインストール＆ログインを一緒に行う
- 先生に基本操作を15分レクチャー
- 生徒への配布方法を決定
  （授業中にApp Storeリンク共有 / LMSに掲示 / メール配信）
```

### Week 1: 生徒への展開
```
- 先生が授業で生徒にLecsyを紹介
- 生徒がApp StoreからLecsyをダウンロード
- 招待リンク or メールアドレス追加で組織に参加
- 最初の授業で録音を体験
```

### Week 2〜4: パイロット運用
```
毎週:
  - 先生にチェックイン（「使い心地どうですか？」）
  - Web管理画面で利用データ確認
    - アクティブユーザー数
    - 録音回数
    - AI機能利用回数
  - 問題があれば即日対応

2週間後:
  - 先生に詳しいフィードバックをヒアリング
  - 生徒の反応を聞く

4週間後:
  - 中間レビュー
  - データをまとめて先生/ディレクターに報告
```

### Week 5〜6: クロージング準備

```
  - 利用者数、録音回数、AI機能利用回数
  - 先生のフィードバック引用
  - 生徒の反応

```

### Week 6〜8: クロージング
```

```

```
  3. 先生に新学期のセットアップ支援
```

## 12. トラブルシューティング（現場で想定されるケース）

| 状況 | 対応 |
|------|------|
| 生徒がログインしても組織に入れない | メールアドレスの確認。pendingレコードのメールと一致しているか確認。大文字小文字の違いに注意 |
| 録音が文字起こしされない | WhisperKitモデルのダウンロード状況を確認。初回はWiFi環境でモデルDLが必要 |
| オフラインで録音データが消えた | クラッシュリカバリー機能で復旧試行。UserDefaultsに永続化されている |
| AI機能が使えない（レート制限） | プランの日次上限に達していないか確認。必要ならプランアップグレードを提案 |
| Web管理画面にアクセスできない | ロールの確認。studentはWeb管理画面の閲覧のみ、操作にはteacher以上が必要 |
| 生徒がアプリを削除してしまった | 再インストール → ログイン → 自動で組織に再参加。Supabaseに同期済みデータは復元される |
| 複数の組織に所属したい | 対応済み。1ユーザーが複数組織に所属可能。設定画面から切り替え |
