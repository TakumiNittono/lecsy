## ① 🌐 Real-time Bilingual Captions

### ユーザーストーリー
> 朝9時、MATH 201の講義。教授が "Let's discuss the Fourier transform..." と話し始める。
> 学生のiPhoneには、英語原文と日本語訳が左右に並んで現れる。専門用語 "Fourier transform" はハイライトされ、タップすると日本語で「フーリエ変換: 時間領域の信号を周波数領域に写像する数学的変換」と表示。
> 教授の早口で聞き取れなかった部分は、"⏪ 30秒戻る" で再表示。

### データフロー

```
iOS Audio (AVAudioSession)
  ↓ 16kHz PCM
Starscream WebSocket to Deepgram
  ↓ English text (word-level timestamp)
Edge Function: translate-realtime
  ↓ GPT-4o Mini streaming (sentence-by-sentence)
Back to iOS (SSE or WebSocket)
  ↓
Dual ScrollView with synchronized scroll
  ↓ 並行
Term Highlight job (GPT-4o Mini)
  ↓ 非同期
Mark terms in UI
```

### エッジケース

- **オフライン時**: ローカル録音のみ、復帰時にPrerecorded + 翻訳バッチ処理
- **ネットワーク不安定**: WebSocket自動再接続、過去30秒分の音声バッファ
- **コードスイッチング**（先生が途中日本語混ぜる）: Deepgram Multilingual autoで追従、翻訳も方向逆転
- **長すぎる単一文**: 30秒で区切って翻訳、その後再接続
- **専門用語の頻出**: Syllabus Keyterm Prompting使用時は認識率90%→98%

## ② 📚 Course Hierarchy

### ユーザーストーリー
> 学期開始前、学生がシラバスPDF（MATH 201の15週分）をlecsyにアップ。GPT-4o Miniが解析:
> - コース: MATH 201 - Calculus II
> - 週: 15週
> - 各週のトピック、Learning Goals
> - 宿題期限、Midterm/Final日程
>
> 学期中、講義を録音すると自動で「Week 3 - Wed」に紐付けられる。

### データモデル

```sql
create table public.semesters (
  id uuid primary key default gen_random_uuid(),
  user_id uuid not null references auth.users(id),
  name text not null, -- 'Fall 2026'
  start_date date, end_date date,
  created_at timestamptz default now()
);

create table public.courses (
  id uuid primary key default gen_random_uuid(),
  semester_id uuid references public.semesters(id),
  user_id uuid not null references auth.users(id),
  code text, -- 'MATH 201'
  title text, -- 'Calculus II'
  instructor text,
  syllabus_pdf_url text,
  syllabus_parsed jsonb, -- GPT-4o Mini抽出結果
  meeting_days text[], -- ['Mon', 'Wed', 'Fri']
  meeting_time text,
  final_exam_date date,
  midterm_dates date[],
  color text -- UI用
);

create table public.weeks (
  id uuid primary key default gen_random_uuid(),
  course_id uuid references public.courses(id),
  week_number int,
  start_date date,
  topic text,
  learning_goals text[],
  assignments jsonb,
  created_at timestamptz default now()
);

create table public.lectures (
  id uuid primary key default gen_random_uuid(),
  week_id uuid references public.weeks(id),
  course_id uuid references public.courses(id),
  user_id uuid not null references auth.users(id),
  transcript_id uuid references public.transcripts(id),
  title text,
  recorded_at timestamptz,
  duration_sec int,
  auto_linked boolean default false, -- 自動紐付け
  manual_tags text[]
);
```

### シラバスPDF自動解析（GPT-4o Mini）

```
User uploads syllabus.pdf
  ↓
Edge Function: parse-syllabus
  ├─ pdf2text抽出
  ├─ GPT-4o Mini プロンプト:
  │   "このシラバスから以下を抽出してJSON: course_code, title, 
  │    instructor, meeting_schedule, 15 weeks with topics,
```

## ③ 🤖 AI Study Guide + Quiz

### 3階層の深度

#### Quick (30秒で読める) — Free Tier 可
```
■ サマリ（3行）
1. フーリエ変換は時間→周波数領域への写像
2. 連続関数とインパルス関数で定義異なる
3. 逆変換で元の信号を復元できる

🇯🇵 日本語版
1. フーリエ変換とは時間領域の信号を周波数領域に変換する手法
2. 連続信号とデジタル信号で定義式が違う
3. 逆変換により元の時間信号に戻せる

📖 キーターム
- Fourier transform (フーリエ変換)
- frequency domain (周波数領域)
- continuous function (連続関数)
- inverse transform (逆変換)
- impulse function (インパルス関数)
```

#### Standard (5分で理解) — Pro
```
■ 完全な要約
（1000-1500単語）

■ 概念マップ
Time Domain ←→ Frequency Domain
  ↓
[visualization]

■ Q&A 10問（Bilingual）
Q1 (英): What is the main difference between continuous and discrete Fourier transforms?
A1: Continuous uses integration... (+ 日本語訳)

■ 前回との繋がり
Week 2では integral の基礎をやりました。
今週のFourier transformはその応用です。

■ 次回予告
Week 4は Laplace transform、Fourier との違いに注目。
```

#### Deep (試験対策) — Pro
```
■ 練習問題 20問 + 詳細解説
Problem 1: Compute F(ω) for f(t) = e^(-at)u(t)
Solution: Step-by-step...
  ↓ 学生が解く
  ↓ 間違い → 解説 + 追加問題

■ Flash Cards（50枚）
Anki/Quizletに即export可

■ Cross-lecture Comparison
Week 3 Fourier vs Week 4 Laplace
  → 共通点・相違点表

■ Weakness Analysis
あなたが苦手: "Integration by substitution"
  → 前回講義の該当箇所を再視聴推奨
  + 追加練習5問
```

### 技術実装

```
Transcript (words[] with timestamps)
  ↓
Edge Function: extract-vocabulary
  ├─ GPT-4o Mini: 「この講義から
  │     大学アカデミック語彙の重要用語を10個抽出、

```

### データモデル

```sql
create table public.personal_vocabulary (
  id uuid primary key default gen_random_uuid(),
  user_id uuid not null references auth.users(id),
  term text not null,
  definition_en text,
  definition_native text, -- 母語訳
  pronunciation text,
  cefr_level text, -- 'A1'-'C2'
  domain text, -- 'STEM', 'Humanities', ...
  embedding vector(1536),
  first_encountered_lecture_id uuid,
  encounter_count int default 1,
  mastered boolean default false,
  last_reviewed_at timestamptz,
  next_review_at timestamptz, -- Spaced Repetition
  created_at timestamptz default now()
);

create index on public.personal_vocabulary
  using ivfflat (embedding vector_cosine_ops);
```

## ⑤ 🎯 Exam Prep Plan

### ユーザーストーリー
> 10月1日、MATH 201 Midterm 1が10月15日に設定されている。
> lecsyが自動通知:
> 「Midterm 1 まで2週間。lecsy AI が学習計画を作成しました」
>
> 以降14日間、毎朝8時にlecsyが「今日の学習タスク」をPushする。

### 自動生成アルゴリズム

```python
def generate_exam_plan(exam_date, course_id):
    days_until = (exam_date - today).days
    
    # Phase分け
    phase_1_broad = 40% of days  # Week 14-10: 全体復習
    phase_2_focused = 35%        # Week 9-4: 弱点強化
    phase_3_active = 20%         # Day 3-1: Active recall
    phase_4_final = 5%           # Day 0: 最終まとめ
    
    # シラバスから試験範囲判定
    covered_lectures = syllabus.midterm_1_scope  # e.g., Week 1-6
    
    # 弱点特定（過去のQuizスコア・Vocabularyレビュー）
    weaknesses = detect_weak_concepts(user_id, course_id)
    
    # Professor Style検出
    prof_style = analyze_past_lectures(course_id)
    # 例: "概念理解重視" / "計算問題重視" / "応用問題重視"
    
    daily_tasks = []
    for day in range(days_until, 0, -1):
        if day in phase_1_broad:
            daily_tasks.append({
                'type': 'quick_summary_review',
                'lectures': covered_lectures[day % len(covered_lectures)],
                'time_minutes': 30
            })
        elif day in phase_2_focused:
            daily_tasks.append({
                'type': 'weakness_drill',
                'concepts': weaknesses,
                'quiz_count': 10,
                'time_minutes': 45
            })
        elif day in phase_3_active:
            daily_tasks.append({
                'type': 'cumulative_quiz',
                'quiz_count': 50,
                'time_minutes': 60
            })
        else:
            daily_tasks.append({
                'type': 'final_flashcards',
                'time_minutes': 20
            })
    
    return daily_tasks
```

## 🎨 全体UX統合原則

### 1. Single-Tap Philosophy
全機能が「ワンタップで起動」。深いナビゲーションなし。

### 2. Bilingual Everywhere
英語と母語を常に併記、または切替タブ1個。

### 3. Context-Aware
どの画面でも「次にやるべきこと」を提示:
- 講義中 → ライブ字幕
- 講義後 → Study Guide生成ボタン
- Study Guide後 → Vocabulary確認
- 試験2週間前 → Exam Plan通知

### 4. Calm Progress
焦らせない。「今週15単語マスター」のような穏やかなフィードバック。

### 5. Fully Offline-Capable
ネットなくても録音と過去データ閲覧はフル動作。
