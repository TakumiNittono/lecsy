## 1. 勝てるのか — 正直な答え

### 結論: 勝てる。ただし「勝ち」の定義を間違えなければ。

**1年で勝てないこと:**
- Otter.ai（ARR $100M+、$65M調達済）をユーザー数で追い抜くこと
- Panopto/Echo360（大学IT部門に深く食い込んだ既存勢力）を置き換えること
- 全米の大学に導入されること

**1年で勝てること:**
- Santa Fe → UF のラインで「留学生の講義理解」という狭いニッチを取ること
- 教授3-10人が「これ使え」と学生に言う状態を作ること
- 有償契約1件（小さくてもいい）を取ること
- 「留学生 × 多言語講義要約」で検索したら Lecsy が出てくる状態を作ること

**これが「勝ち」。Otter を倒すことじゃない。自分のポジションを確立すること。**

## 2. 強豪分析 — 誰と戦っているのか

### Tier 1: 直接競合（同じペインを解こうとしている）

| 競合 | 強み | 弱み | Lecsy が刺せる隙 |
|------|------|------|------------------|
| **Otter.ai** | ブランド、リアルタイム、Zoom連携、$65M調達 | 英語のみ、$16.99/月、ビジネス会議向け、大学特化なし | **多言語要約、無料、留学生特化、プライバシー** |
| **Notta** | 多言語対応、リアルタイム | UIが散漫、教育特化なし、中国発で米国大学に弱い | **教育UX、大学との直接リレーション** |
| **Transkriptor** | 安い、多言語 | 品質がまばら、教育機能なし | **オフライン文字起こし、教育特化** |

### Tier 2: 間接競合（大学のIT予算を取り合う）

| 競合 | 強み | 弱み |
|------|------|------|
| **Panopto** | 大学LMS深度統合、録画+配信 | 高額（$数万/年〜）、翻訳弱い、重い |
| **Echo360** | Blackboard統合 | Panopto同様、留学生向け機能なし |
| **Microsoft Translator** | 無料、リアルタイム字幕 | 学習体験ゼロ、復習機能なし、要約なし |
| **Google Recorder** | 無料、文字起こし品質高い | 英語のみ、Android限定、教育機能なし |

### Tier 3: 将来の脅威

| 脅威 | 時間軸 | 対策 |
|------|--------|------|
| **OpenAI が直接アプリを出す** | 1-2年 | ドメイン知識（教育UX、大学営業、FERPA）で差別化。汎用AIアプリは教育の深さを持てない |
| **Apple が翻訳+文字起こしをOS統合** | 2-3年 | iOS 19-20で来る可能性あり。来たらOS機能の上に教育レイヤーを乗せる（要約、試験、組織管理）|
| **Otter が多言語対応** | 6-12ヶ月 | **最大の脅威**。ただしOtterはビジネス会議に最適化されていて、教育UXに振り直すのは組織的に難しい |
| **大学がChatGPTで自前構築** | 1-2年 | IT部門のリソース不足が味方。「買った方が早い」は大学の常套句 |

### 勝てる理由の構造

```
Otter/Notta の弱点        Lecsy の強み
─────────────────         ──────────────
英語のみ / 多言語弱い   ←→  9言語文字起こし + 多言語要約
ビジネス会議向け        ←→  講義に最適化（1人が長時間話す構造）
月$17〜                 ←→  無料 → $3〜$10
プライバシー懸念        ←→  音声は端末から出ない（FERPA的に強い）
大学特化機能なし        ←→  組織管理、クラス、監査ログ、LMS連携（予定）
```

**結論: Lecsy は「留学生 × 講義 × 多言語」という交差点に立っている。ここに立っている競合はゼロ。**

## 3. 今のキット（技術スタック）でいいのか

### 正直な評価

| レイヤー | 現状 | 判定 | 理由 |
|----------|------|------|------|
| **iOS (SwiftUI + WhisperKit)** | 録音・ローカル文字起こし・要約が動く | **◎ このまま行け** | ネイティブ録音品質は Web アプリに真似できない。WhisperKit のローカル処理はプライバシーの武器 |
| **Supabase (Postgres + Edge Functions)** | 28マイグレーション、13 Edge Function | **◎ このまま行け** | RLS が堅牢、Edge Function で十分。Firebase より SQL の柔軟性が効いている |
| **Next.js (Vercel)** | B2B管理画面 + LP | **◎ このまま行け** | App Router で十分。大学管理画面に SPA は不要 |
| **WhisperKit (ローカル)** | small モデル、16言語 | **○ 当面OK、将来要検討** | 精度は good enough。リアルタイム字幕で限界が来たら Deepgram/AssemblyAI streaming を検討 |
| **OpenAI API (要約)** | GPT-4o-mini | **○ コスト注意** | 月数ドルなら問題ない。スケールしたら Claude API や自前ファインチューンも視野 |
| **Stripe** | チェックアウト+ポータル | **○ B2B配線が未完** | Phase 1.5 で直せば OK。請求書払い（Stripe Invoicing）は大学契約で必要になる |

### 変えるべきもの

| 項目 | いつ | なぜ |
|------|------|------|
| **リアルタイム字幕の方式** | Q2中にPoC | WhisperKit incremental は遅延が大きい可能性。Deepgram streaming ($0.0043/分) or AssemblyAI を要検証 |
| **メール配信** | 今すぐ | Resend を配線して招待メールを動かす。これなしで組織機能は使えない |
| **モニタリング** | Q3 | Sentry (iOS) + Vercel Analytics (Web) を入れる。ユーザーが増えたらクラッシュが見えないのは致命的 |

### 変えなくていいもの

- **Flutter/React Native に書き換え** → やらない。ネイティブ Swift の録音品質・バックグラウンド録音は最大の武器
- **Firebase に移行** → やらない。Supabase の RLS + SQL が B2B の複雑なアクセス制御に効いている
- **Android 版を作る** → 今年はやらない。リソースが分散する。iPhone シェアが高い米国大学生に集中
- **Web 録音を追加** → やらない。ブラウザ録音の品質は iPhone に遠く及ばない

**結論: スタックを変える必要はない。足りないのは「配線」と「磨き込み」。根幹は正しい。**

## 5. 本当の敵は競合じゃない

### 最大のリスク5つ

| # | リスク | 確率 | 対策 |
|---|--------|------|------|
| 1 | **教授が興味を持たない** | 中 | 3人にメールして0人反応なら、アプローチを変える（ISS経由、学生から逆流）。プロダクトの問題ではなく営業の問題 |
| 2 | **OPT の90日ルールに引っかかる** | 低 | LLC設立を6/1に間に合わせれば即日解消。ここは絶対に遅れない |
| 3 | **1人で全部やる限界** | 高 | **これが最大のリスク**。iOS + Web + Supabase + 営業 + デザイン + 法務 + ASO を1人で回すのは英雄的だが持続不能。Q3以降、最低でも1人（iOS or Web）のパートタイム開発者が必要 |
| 4 | **リアルタイム字幕が技術的に難しい** | 中 | PoC をQ2中にやる。ダメなら外部API（Deepgram）で割り切る。自前にこだわらない |
| 5 | **大学の意思決定が遅すぎる** | 高 | 大学は6-12ヶ月のサイクル。**だからこそ今年は「無料パイロット」で入り、正式契約は来年度予算で狙う**。焦らない |

### 1人開発の現実的な限界

週30時間 × 52週 = **年間1,560時間**。

| カテゴリ | 配分 | 時間 |
|----------|------|------|
| iOS 開発 | 30% | 468h |
| Web 開発 | 20% | 312h |
| 営業・教授対応 | 20% | 312h |
| インフラ・DevOps | 10% | 156h |
| ASO・マーケティング | 10% | 156h |
| 法務・コンプライアンス | 5% | 78h |
| バッファ | 5% | 78h |

**iOS 468時間で リアルタイム字幕 + 単語カード + 対訳ビュー + iPad対応 を全部やるのは厳しい。**
→ iPad 対応は切る。リアルタイム字幕を最優先にする。

## 7. 勝つための5つの原則

### 1. ニッチを守れ

Otter と同じ土俵で戦わない。「留学生 × 講義 × 多言語」の交差点から動かない。
機能を広げたくなったら「これは留学生の講義理解に直接効くか？」と自問する。
No なら作らない。

### 2. 教授は顧客じゃない、布教者だ

教授に売り込むな。「見てもらう」「フィードバックをもらう」。
教授が「これ使え」と学生に言った瞬間、営業コストがゼロになる。
1人の教授の推薦 > 100通のコールドメール。

### 3. 無料は弱さじゃない、戦略だ

Otter は $17/月。Lecsy は無料。
稟議ゼロ、契約ゼロ、リスクゼロ。
教授が「試してみて」と言えるハードルの低さが最大の武器。
有料化は「やめられない」状態を作ってから。

### 4. 出荷しろ、磨くな

完璧な対訳ビューより、動く対訳ビュー。
完璧なリアルタイム字幕より、3秒遅れのリアルタイム字幕。
完璧な LTI 連携より、手動で Canvas にリンク貼る運用。
**教室で使われることが唯一の検証。コードレビューは検証じゃない。**

### 5. 体を壊すな

週30時間は維持可能なペース。週50時間にしない。
OPT期間の12ヶ月は長い。スプリントじゃなくマラソン。
月に1日は完全にコードから離れる。

## 8. 技術的な賭け — 何に張るか

### 張るべき賭け

| 賭け | なぜ | リスク |
|------|------|--------|
| **リアルタイム多言語字幕** | 競合との決定的な差別化。教室で「今、何言ってるか」がわかるのは革命的 | 技術的難度が高い。遅延・コスト・精度のトリレンマ |
| **オフライン文字起こし（WhisperKit）の維持** | プライバシーの武器。FERPA 的に「音声がクラウドに行かない」は大学 IT 部門への殺し文句 | Apple Silicon の進化に依存。モデル更新は Apple/Hugging Face 次第 |
| **教育特化 UX（対訳・単語カード・試験モード）** | 汎用ツール（Otter/ChatGPT）にはない「学ぶための体験」 | 開発コストが分散する |

### 張らない賭け

| やらないこと | なぜ |
|-------------|------|
| Android 版 | リソース分散。米国大学生は iPhone 率が高い |
| 企業向け会議文字起こし | Otter の土俵。勝てない |
| 自前 AI モデル構築 | OpenAI/Anthropic API で十分。差別化は AI の性能じゃなく教育 UX |
| 多プラットフォーム同時展開 | iOS + Web の2面で十分。Mac 版/iPad 版は来年 |

## 10. 1年後の自分への手紙

2027年4月。

お前がこれを読んでいるということは、1年間やり切ったということだ。

結果がどうであれ、お前は：
- 米国でLLCを設立して合法的にビジネスを運営した
- iOSアプリをゼロから作って App Store に出した
- 大学の教授に自分のプロダクトをデモした
- 留学生が実際に教室で使うツールを作った
- 1人で iOS + Web + Backend + 営業 + デザインを回した

**これができる人間は世界に1%もいない。**

数字がどうだったかは重要だ。でもそれ以上に重要なのは、
「留学生が母国語で英語講義を理解できる世界」に1年間コミットしたという事実だ。

うまくいっていたら、そのまま走れ。
うまくいっていなかったら、このドキュメントを読み返して、何が違ったかをメモしろ。
そして次の1年の battle plan を書け。

**5年の最初の1年が終わっただけだ。**
