### UX / 運用改善
- **Sign-out 60秒 UI 無反応**: Supabase /logout が -1005 QUIC バグで詰まる → ローカル state 即クリア + サーバ側 revocation は fire-and-forget
- **signedOut イベント storm**: 一時的な refresh 失敗で SDK が signedOut を 20+ 回連発 → 30秒 cooldown でデバウンス
- **ホーム画面の "Preparing AI model..."** を削除して裏で silent に準備
- **Deepgram keep-warm**: ホーム滞在中に 4分毎 preconnect 再接続、言語切替 / foreground 復帰でも再 preconnect
- **Summary streaming の first paint**: `SupabaseClient.streamFunction` のバッファ 256B → 16B、体感 3-5秒 → 1-2秒
- **On-Device トグル消失バグ**: `isPaid` ガードが自己無効化 → `isProEntitled` 分離
- **Login 画面コンパクト化**: 上端 60pt / 中段 24pt の Spacer 削除、mic アイコン 88→44、タイトル 48→30、全ボタン高さ 50→44、横 padding 48→40 で約 180pt 短縮
- **Google / Apple ボタンの幅バラつき**: 全ボタンを 375pt 上限 + 中央寄せに統一 (Apple HIG 準拠)
- **オフライン表示**: ネット断時にホーム画面上部に小さな橙ドット + "Offline — recording saves locally"
- **FERPA PATCH の -1005 死**: stale-socket retry 追加
- **classes UI の 404 spam**: `loadRealClasses` を no-op 化 (`org_classes` drop 済)
- **音声の自動削除設定** (opt-in): Settings → Privacy で 30/90/180 日 / Forever (デフォルト) 選択可、transcript は残す

## 3. 💾 データ保存先 (現時点の正確な全体図)

### iOS 端末内のみ (cloud に絶対出ない)
| 対象 | パス | 保護 |
|---|---|---|
| 音声 (.m4a) | `Documents/recording_<timestamp>.m4a` | `completeUntilFirstUserAuthentication` + `isExcludedFromBackup` |
| Lecture メタ (transcript / ownerUserID / pendingCloudSync / cachedSummary / bookmarks) | `Documents/lectures.json` (+ `_backup.json`) | 同上 |
| Auth トークン | Keychain | `kSecAttrAccessibleAfterFirstUnlockThisDeviceOnly` (iCloud 復元で移行しない) |
| 設定 (言語 / cloudSyncEnabled / 音声リテンション / 同意キャッシュ) | UserDefaults | iOS 標準 |

### Supabase に保存 (text のみ、音声は一切書かない)
| テーブル | いつ書く | 内容 |
|---|---|---|
| `transcripts` | Deepgram 経路保存時のみ | text / duration / language / user_id / organization_id / visibility |
| `usage_logs` | 要約ボタン押下時 | user_id / action='summarize' |
| `summaries` | 要約生成成功時 | sections / key_points |
| `organization_members` | 招待受諾 / FERPA 同意 | membership + consented_at |
| `organizations` | org 作成 | 組織名 / plan / 学校名 |
| `auth.users` | Supabase Auth が管理 | email / user_id |
| `audit_logs` | B2B 操作 | 招待 / 権限変更 / 削除履歴 |
| `org_ai_usage_logs` | 組織横断 AI (cross-summary / glossary) | user_id / action |

### 第三者経由 (保管はしない)
| 宛先 | 送るもの | 扱い |
|---|---|---|
| Deepgram | 音声ストリーム | リアルタイム文字起こし、セッション終了で discard |
| OpenAI | transcript text | 要約のみ、OpenAI zero-retention |
| Sentry | error / trace event | PII 最小 (IP off、email のみ)、Replay は mask 済 |

### 絶対に保存されないもの
- **音声が cloud に行く経路はゼロ** (Otter / Notta との差別化の核心)
- **WhisperKit の transcript は cloud に行かない**
- **Anonymous 録音は cloud に行かない**

## 4. 🔒 プライバシー / セキュリティ姿勢

### 今のレベルで耐える脅威
| 脅威 | 防御 |
|---|---|
| 端末紛失 → 拾った人 | iOS passcode + ファイル保護 + Keychain device-only |
| 家族で iPad 共有 | Library が `ownerUserID` で user-scoped、サインアウト中は自分の分しか見えない |
| 教室共有端末 | 同上 + "Sign Out & Delete Data on This Device" オプション |
| 通信傍受 | TLS 1.2+ 全経路 |
| Supabase DB 直叩き | RLS (`user_id = auth.uid()` 強制) |
| トークン漏洩 | Keychain / 1h TTL / サーバ revocation |
| 別ユーザーアカウント誤 upload | 自動 backfill 廃止済、orphan 51行削除済 |
| ネット断での録音消失 | pending queue で復帰後自動再送 |
| 音声の cloud 漏洩 | **そもそも cloud に送信していない** |

### Otter / Notta 比較
- **Otter**: raw audio をサーバに保管 → 集団訴訟中
- **Notta**: 音声 + transcript、データ越境
- **Lecsy**: 音声は端末から出ない、transcript も明示サインインした org にしか出ない

### FERPA posture
- 音声非保持 = FERPA の "educational record" 該当範囲を最小化
- org member 登録時に `ferpa_consented_at` 明示スタンプ
- DPA + Addendum 用意 (founder@lecsy.app 請求制)
- 保管: US AWS us-east-1 (Supabase 管理)、Retention: transcript 手動削除まで、usage_logs 90日
