# Lecsy 統合要件定義書 v3（B2B 進化版・iOS + Web 連結）

**作成日:** 2026-04-06
**ステータス:** 確定版（v2 を継承し B2B を進化）
**継承元:** `doc/00_統合要件定義_v2.md`
**置き換え対象:** v2 の §1.2「Web 廃止」、§5.1「Web 削除」、§8/§9 の削除リスト

> **方針アップデート:** v2 では Web を最小化する方針だったが、現状の `web/app/org/**` が既に動作しており、学校の事務職員が PC ブラウザで一覧・出力したい現実的ニーズも強い。v3 では **iOS をプライマリ操作面、Web をセカンダリ閲覧/重作業面とする「2 面 1 バックエンド」構成**に進化させる。
>
> **ベースとフローは v2 のまま**:
> - メール一致による pending → active 自動昇格
> - スーパー管理者のみが owner 権限を付与
> - 招待リンク方式は復活させない
> - 1 ユーザー複数組織対応
> - DB マイグレーション順序 M6→M5→M1〜M4 は据え置き

## 1. 2 面 1 バックエンドの責務分担

### 1.1 「iOS でしかできないこと」と「Web の方が向いていること」を分ける

| 機能領域 | iOS | Web | 理由 |
|---|:---:|:---:|---|
| 録音・文字起こし | ✅ 唯一 | ❌ | WhisperKit はオンデバイス |
| 個人ライブラリ閲覧 | ✅ | ✅(read) | 出先と席で使う |
| AI 要約生成 | ✅ | ✅ | どちらでもトリガー可 |
| **メンバー追加（少数）** | ✅ | ✅ | 営業現場は iPad、事務は PC |
| **メンバー一括追加（CSV 100 名+）** | △ | ✅ 唯一 | ファイル UI は Web 向き |
| クラス管理 | ✅ | ✅ | |
| 用語集（閲覧/編集） | ✅ | ✅ | |
| 用語集（生成） | ✅ | ✅ | |
| Cross-Summary | ✅ | ✅ | |
| 利用統計（カード/表） | ✅ | ✅ | |
| **利用統計（グラフ・期間比較）** | △ | ✅ 主 | 大画面の方が分析しやすい |
| **CSV / PDF レポート出力** | ❌ | ✅ 唯一 | ファイル DL は Web 向き |
| **Stripe 課金（B2B）** | リンクのみ | ✅ 主 | Reader app 規定回避 |
| 組織設定（名前/タイプ） | ✅ | ✅ | |
| **スーパー管理者操作** | ✅ | ✅ | iPad/PC どちらからでも |
| 監査ログ閲覧 | △ 直近のみ | ✅ 主 | 期間検索は Web |
| **メール自動送信トリガー** | ❌ | ✅ 唯一 | サーバアクションは Web |

**設計原則:**
1. **書き込みは両面で対応可、ただし重い操作（CSV 取込・PDF 出力・期間検索）は Web のみ**
2. **iOS と Web は同じ Supabase + 同じ RLS を見る**。新規 API は作らず Edge Function を共用
3. UI 系統は 2 つだが、**サービス層と権限ロジックは 1 つ**（DB 関数 + Edge Function）
4. iOS のリリースサイクルに引きずられないよう、緊急修正は **Web 側でも完結**できるようにする

### 1.2 「やらないこと」（v2 から維持）

- ❌ Web 単独の B2B 専用アプリ化
- ❌ Web で個人録音を録る
- ❌ 招待リンク方式の復活
- ❌ Web で owner 権限を一般ユーザーに付与する経路
- ❌ App Store に B2B 専用アプリを別出し

## 2. Web 側の正式スコープ（v3 で確定）

### 2.1 残す/育てるルート

```
web/app/
├── page.tsx                          # LP（B2C/B2B 両立）
├── login/                            # Supabase Auth UI
├── auth/callback                     # OAuth コールバック
├── privacy/, terms/                  # 審査用
├── app/                              # ★個人ダッシュボード（read 中心、編集は最小）
│   ├── page.tsx                      # 録音一覧
│   └── t/[id]                        # 個別録音（テキスト閲覧/AI要約再生成）
├── org/                              # ★B2B 管理面（v3 の主役の一つ）
│   ├── new                           # 組織作成（superadmin only）
│   └── [slug]/
│       ├── page.tsx                  # 組織ダッシュボード（KPI カード + グラフ）
│       ├── members/                  # 一覧 + 単発追加 + ロール変更 + 削除
│       ├── members/import            # ★ CSV 一括追加（Web 専用）
│       ├── classes/                  # クラス CRUD
│       ├── glossary/                 # 用語集 CRUD + 生成
│       ├── usage/                    # 利用統計（期間切替・グラフ・CSV/PDF）
│       ├── ai/                       # Glossary / Cross-Summary 生成 UI
│       ├── audit/                    # ★ 監査ログ閲覧（Web 専用）
│       ├── settings/                 # 組織設定
│       └── billing/                  # ★ Stripe Checkout / Customer Portal
├── admin/                            # ★ スーパー管理者ダッシュボード
│   ├── page.tsx                      # 全組織一覧
│   ├── orgs/new                      # 組織作成
│   └── orgs/[id]                     # 組織ステータス管理・owner 付与
└── api/
    ├── org/[slug]/...                # 既存。RLS + Edge Function 経由
    ├── org/[slug]/members/import     # CSV パース + バルク INSERT
    ├── org/[slug]/usage/export       # CSV/PDF 出力
    ├── org/[slug]/billing/checkout   # Stripe Checkout Session
    ├── org/[slug]/billing/portal     # Stripe Customer Portal
    └── activate-membership/route.ts  # 既存維持
```

### 2.2 認証と認可（Web）

- 既存 Supabase Auth セッションをそのまま使う
- **全ての `/org/[slug]/**` ルートは Server Component 側で `requireOrgRole(slug, minRole)` を呼ぶ**
- `requireOrgRole` は SECURITY DEFINER の `is_org_role_at_least(org_slug, uid, role)` を実行（v3 で新設）
- middleware で `/org/**` と `/admin/**` を未ログイン時 `/login` リダイレクト

### 2.3 Web デザイン原則

- 既存 Tailwind デザイントークン継承
- iPad Safari でも崩れないレスポンシブ（営業現場で iPad ブラウザを使うケース）
- データテーブルは shadcn/ui の DataTable 相当 + ソート/ページング
- グラフは Recharts（軽量・SSR 可）
- ダーク/ライト両対応

## 3. iOS 側の責務（v2 を継承し B2B を進化）

### 3.1 v2 から変更しない部分

- タブ構成（個人 / 組織所属でロール出し分け）
- ログイン直後の `activate_pending_memberships` 呼び出し
- スーパー管理者画面（iPad で営業現場の組織作成）
- RecordView / LibraryView の `organization_id` / `visibility` 付与

### 3.2 v3 で iOS に追加する B2B 機能

| # | 機能 | 概要 |
|---|---|---|
| iOS-1 | **組織ホームの実装強化** | KPI カード（今日/今週/今月の録音数・アクティブ生徒・AI 利用回数）+ 直近 10 件アクティビティ |
| iOS-2 | **「Web を開く」深いリンク** | 「PDF レポートを出力」「CSV 取込」など Web でしか出来ない操作には `SFSafariViewController` で該当 Web ルートに遷移するボタンを置く |
| iOS-3 | **オフラインキューイング** | 組織共有録音は WiFi 復帰時にバルクアップロード、進捗を組織ホームに表示 |
| iOS-4 | **クラス実体への録音バインド** | RecordView 上部で「現在のクラス」をピン止め可能。録音は自動で `class_id` 付与 |
| iOS-5 | **生徒向け学習通知** | 組織所属時、新しい用語集が追加されたら Push 通知（学校が ON にした場合のみ） |
| iOS-6 | **講師向け教室モード** | 1 タップで「録音 + 全生徒に共有 + リアルタイム文字起こしを大画面に映す（外部ディスプレイ対応）」 |
| iOS-7 | **Live Activity / Dynamic Island** | 録音中ステータス（既存 LecsyWidgetExtension を組織モードに拡張） |
| iOS-8 | **iPad 最適化レイアウト** | サイドバー + 詳細ペインの NavigationSplitView。営業現場・教室の iPad で映える |

### 3.3 iOS の Web 連携リンクリスト

| iOS 画面 | 「Web で開く」ボタン | 遷移先 |
|---|---|---|
| 組織ホーム | レポート作成 | `/org/[slug]/usage?export=pdf` |
| メンバー一覧 | CSV で一括追加 | `/org/[slug]/members/import` |
| 利用統計 | 詳細グラフを見る | `/org/[slug]/usage` |
| 課金（owner） | プラン変更 | `/org/[slug]/billing` |
| スーパー管理者 | 全組織を一覧 | `/admin` |

すべて **Magic Link なしに seamless にログイン状態を引き継ぐ**ため、`SFSafariViewController` で Supabase Cookie を共有する。または iOS から `supabase.auth.signInWithIdToken()` の token を取り、`https://lecsy.app/auth/callback?token=...&next=/org/...` 形式で渡す。

## 4. データモデル（v2 + v3 追記）

### 4.1 v2 で確定済みマイグレーション（変更なし）

M6 → M5 → M1 → M2 → M3 → M4（v2 §4.2 参照）

### 4.2 v3 追加マイグレーション

#### M7. organizations 拡張
```sql
ALTER TABLE organizations
  ADD COLUMN allowed_email_domains TEXT[] DEFAULT '{}',
  ADD COLUMN logo_url TEXT,
  ADD COLUMN locale TEXT NOT NULL DEFAULT 'en',
  ADD COLUMN timezone TEXT NOT NULL DEFAULT 'UTC',
  ADD COLUMN settings JSONB NOT NULL DEFAULT '{}'::jsonb;
```
- `allowed_email_domains`: 空配列なら制限なし。値があればそのドメイン以外の pending 作成を RLS で拒否
- `settings`: `{ recording_default_visibility, monthly_recording_cap_per_user, push_notifications, ... }`

#### M8. classes に学期 + 期間
```sql
ALTER TABLE org_classes
  ADD COLUMN start_date DATE,
  ADD COLUMN end_date DATE,
  ADD COLUMN archived BOOLEAN NOT NULL DEFAULT false;

CREATE INDEX idx_org_classes_active ON org_classes(org_id) WHERE NOT archived;
```

#### M9. 録音 → クラスのバインド
```sql
ALTER TABLE transcripts
  ADD COLUMN class_id UUID NULL REFERENCES org_classes(id) ON DELETE SET NULL;

CREATE INDEX idx_transcripts_class ON transcripts(class_id) WHERE class_id IS NOT NULL;
```
RLS: 同クラスメンバーは visibility='class' の transcript を閲覧可。

#### M10. 監査ログの拡張インデックス
```sql
CREATE INDEX idx_audit_actor_time ON audit_logs(actor_user_id, created_at DESC);
CREATE INDEX idx_audit_action_time ON audit_logs(action, created_at DESC);
```

#### M11. 招待ドメイン制限の RLS
```sql
CREATE OR REPLACE FUNCTION email_domain(p_email TEXT)
RETURNS TEXT LANGUAGE sql IMMUTABLE AS $$
  SELECT lower(split_part(p_email, '@', 2));
$$;

-- members_insert_admin を強化
DROP POLICY IF EXISTS "members_insert_admin" ON organization_members;
CREATE POLICY "members_insert_admin" ON organization_members FOR INSERT WITH CHECK (
  (user_id = (SELECT auth.uid()))
  OR EXISTS (
    SELECT 1 FROM organization_members AS om
    JOIN organizations o ON o.id = om.org_id
    WHERE om.org_id = organization_members.org_id
      AND om.user_id = (SELECT auth.uid())
      AND om.role IN ('owner', 'admin')
      AND (
        coalesce(array_length(o.allowed_email_domains, 1), 0) = 0
        OR email_domain(organization_members.email) = ANY(o.allowed_email_domains)
      )
  )
);
```

#### M12. ロール判定ヘルパ（Web/Edge Function 共用）
```sql
CREATE OR REPLACE FUNCTION is_org_role_at_least(p_slug TEXT, p_user UUID, p_min TEXT)
RETURNS BOOLEAN LANGUAGE sql SECURITY DEFINER STABLE AS $$
  WITH ranks AS (
    SELECT 'student'::TEXT AS r, 0 UNION ALL
    SELECT 'teacher', 1 UNION ALL
    SELECT 'admin',   2 UNION ALL
    SELECT 'owner',   3
  )
  SELECT EXISTS (
    SELECT 1
    FROM organization_members om
    JOIN organizations o ON o.id = om.org_id
    JOIN ranks r1 ON r1.r = om.role
    JOIN ranks r2 ON r2.r = p_min
    WHERE o.slug = p_slug
      AND om.user_id = p_user
      AND om.status = 'active'
      AND r1.column2 >= r2.column2
  );
$$;
```

#### M13. Stripe webhook 用カラム
```sql
ALTER TABLE organizations
  ADD COLUMN stripe_subscription_status TEXT,
  ADD COLUMN current_period_end TIMESTAMPTZ,
  ADD COLUMN cancel_at_period_end BOOLEAN NOT NULL DEFAULT false;
```

## 5. Edge Functions（共用ロジック層）

| Function | 用途 | 呼び出し元 |
|---|---|---|
| `summarize` | 個人/組織の AI 要約 | iOS / Web |
| `org-ai-assist` | Glossary 生成 / Cross-Summary | iOS / Web |
| `save-transcript` | 録音保存 | iOS のみ |
| `stripe-webhook` | Stripe イベント受信 | Stripe |
| **`org-create`**（新規 v3） | スーパー管理者のみ組織作成 | iOS / Web 両方 |
| **`org-grant-ownership`**（新規 v3） | スーパー管理者のみ owner 付与 | iOS / Web 両方 |
| **`org-checkout`**（新規 v3） | Stripe Checkout Session 作成 | Web 主、iOS は SafariView |
| **`org-csv-import`**（新規 v3） | CSV パース + メンバー一括追加 | Web のみ |
| **`org-report-export`**（新規 v3） | 利用統計 PDF/CSV 生成 | Web のみ |
| **`org-send-notification`**（新規 v3） | Resend 経由のメール配信 | Web のみ |

すべての新規 Function は **`is_super_admin()` または `is_org_role_at_least()` を最初に呼んで権限チェック**する。Web/iOS のどちらから来ても結果は同じ。

## 7. Web UI の進化ポイント詳細

### 7.1 組織ダッシュボード `/org/[slug]`

- 上部: KPI カード x4（iOS と同じ指標、ただし期間切替付き）
- 中段: 録音数の日次推移グラフ（折れ線、Recharts）+ 言語別円グラフ
- 下段: メンバー別ランキング表（録音数 desc）
- 右サイドバー: 直近 20 件のアクティビティ
- ヘッダーに「PDF 出力」「CSV 出力」ボタン

### 7.2 CSV 一括追加 `/org/[slug]/members/import`

- ドラッグ&ドロップで CSV をアップロード
- ヘッダー必須: `email`, `role`（任意で `display_name`, `class`）
- プレビュー画面で問題行をハイライト（重複/不正メール/座席超過/ドメイン違反）
- 「修正可能なものだけ追加」ボタン
- 結果: 成功 N 件 / 失敗 M 件のサマリ + 失敗 CSV ダウンロード

### 7.3 利用統計 `/org/[slug]/usage`

- 期間: 今週 / 今月 / 先月 / カスタム
- 比較: 前期間との差分（% 表示）
- メンバー別表 + 各列ソート
- 「PDF レポート」「CSV エクスポート」「Slack/Email で送信」ボタン
- 学校に渡せる体裁の PDF（ロゴ + 期間 + KPI + グラフ + メンバー表）

### 7.4 監査ログ `/org/[slug]/audit`

- 期間 + アクター + アクションタイプでフィルタ
- 結果は仮想スクロール
- CSV エクスポート
- コンプライアンス監査時に提出可能な形式

### 7.5 課金 `/org/[slug]/billing`

- 現プラン表示 + 次回請求日 + 座席使用率
- プラン変更（Stripe Checkout に遷移）
- 請求履歴 + Stripe Customer Portal リンク
- アップグレード比較表

### 7.6 スーパー管理者 `/admin`

- 全組織テーブル（名前・slug・plan・seats・MRR・最終アクティビティ）
- 「新規組織」ボタン → `/admin/orgs/new`
- 各組織の「owner を付与」ダイアログ（メアド入力）
- 全体 KPI（総組織数・MAU・MRR・新規追加組織）

## 付録 A. iOS ↔ Web シングルサインオン（SFSafariViewController）

iOS から Web に遷移する際、再ログインを要求しないために：

1. iOS で `supabase.auth.session?.access_token` を取得
2. Web 側に `https://lecsy.app/auth/exchange?token=...&next=/org/ohla/usage` を作る
3. exchange ルートが Cookie を Set してリダイレクト
4. SFSafariViewController で開くと Cookie は Safari と独立だが、Universal Link で本物の Safari に渡せば共有される

または ASWebAuthenticationSession を使い、専用のセッションコンテキストで Cookie を共有する方式も検討。

## 付録 B. Web → iOS への深いリンク

Web の組織ダッシュボードに「iOS で録音を開始」ボタンを置き、Universal Link `https://lecsy.app/record?org=ohla&class=...` で iOS アプリを起動する。アプリは該当組織/クラスをアクティブにして RecordView を開く。

## 付録 C. 障害時のフェイルオーバー

- iOS 単独で動作可能（Web が落ちていても録音・文字起こし・AI 要約は動く）
- Web 単独でも動作可能（iOS リリース審査中でも事務作業は止まらない）
- Supabase が落ちると両方止まる → 録音はローカルキューイングで継続
