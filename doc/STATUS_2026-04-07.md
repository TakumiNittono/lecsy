| `OrganizationService` / `OrganizationAPI` | B2B 組織のメンバーシップ取得 |
| `SummaryService` | `/summarize` Edge Function を呼ぶ。**B2B メンバー限定** |
| `LectureStore` | ローカル永続化 (FileManager + UserDefaults) |
| `KeychainService` | access / refresh トークン保管 |

## 4. Supabase

### 4.1 主要テーブル
`transcripts`, `summaries`, `usage_logs`, `organizations`, `organization_members`, `classes`, `glossary`, `audit_logs`, `reports`
すべて RLS で `user_id` / `organization_id` 隔離。

### 4.2 本日追加した未コミットマイグレーション 2 本

**`20260407200000_transcripts_client_id.sql`**
- `transcripts.client_id` (uuid) を追加
- `(user_id, client_id)` の **部分 UNIQUE INDEX** (`WHERE client_id IS NOT NULL`)
- iOS 側 `Lecture.id` を流すことで backfill リトライが冪等に
- 既存行は `NULL` のまま (後方互換)

**`20260407210000_db_hardening.sql`** (5.8 KB)
1. `(user_id, title, created_at)` で重複している `client_id IS NULL` 行を片寄せ削除
2. インデックス追加
   - `transcripts(user_id, created_at DESC)`
   - `transcripts(organization_id, created_at DESC)`
   - `summaries(transcript_id)`
   - `summaries(user_id, created_at DESC)`
   - `usage_logs(user_id, created_at DESC)`
3. `transcripts.updated_at` カラム + `BEFORE UPDATE` トリガ
4. **ON DELETE CASCADE**: `auth.users` 削除時に `transcripts` / `summaries` / `usage_logs` を連鎖削除 (GDPR 対応)
5. `NOT NULL` 制約 (`user_id`, `transcript_id` 等)
6. `CHECK` 制約: `visibility IN ('private','org_wide')`、`duration >= 0`

> 次アクション: `supabase db push` (本番) → コミット → push。

## 5. 認証フロー

### 5.1 サインイン経路
1. **Apple Sign In** — `ASWebAuthenticationSession` + Supabase + Keychain
2. **Google Sign In** — `ASWebAuthenticationSession` + Supabase + Keychain
3. **Magic Link OTP (新規)** — メール送信 → 6 桁コード → `verifyOTP()` → セッション。国際留学生・B2B CSV 招待向け
4. **Skip Login** — ローカルのみ、Cloud Sync は OFF、B2B 機能は使えない

### 5.2 サインイン直後のオーケストレーション
```
checkSession()
 └→ refreshOrgMembership()      // organization_members を query
     └→ isOrgMember = true なら B2B 機能解放
 └→ CloudSyncService.backfillAllLocalLectures()
     └→ 未同期ローカル録音を /save-transcript に送る (client_id で冪等)
     └→ Settings に進捗表示
```
