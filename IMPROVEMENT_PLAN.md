# Lecsy 改善計画書

> 目標: 「人から必要とされるアプリ」にするための包括的な改善提案
> 現状: App Store申請済み / Web版Stripeはライブだが実質テスト段階

---

## 1. 最優先で改善すべき致命的な問題

### 1-1. 初回体験（FTUE）が壊滅的に悪い

**現状の問題:**
- 初回録音後、WhisperKitモデルのダウンロードに5分以上かかる
- ユーザーは「壊れてる？」と思って離脱する
- ダウンロード進捗が見えない

**改善案:**
```
[現状]
録音停止 → 突然ダウンロード開始 → 5分待ち → 文字起こし

[改善後]
初回起動時にオンボーディング画面で事前説明
→ Wi-Fi接続時に裏でモデルを事前DL（オプトイン）
→ DL進捗バー付き（「あと約3分」表示）
→ DL完了通知
→ 録音開始時にはすぐ文字起こし可能
```

**実装の方向性:**
- `lecsyApp.swift` の起動時に `TranscriptionService.loadModel()` をバックグラウンド実行
- オンボーディング画面（3ステップ）を追加:
  1. 「講義を録音」（マイクアニメーション）
  2. 「AIが自動で文字起こし」（テキスト生成アニメーション）
  3. 「どこでも読み返せる」（iPhone↔Web連携イラスト）
- モデルDLの進捗をSettingsにも表示

### 1-2. 英語しか対応していない（最大の機会損失）

**現状の問題:**
- `TranscriptionLanguage` が英語のみ
- 日本の大学生、中国の留学生など巨大市場を逃している
- SEOページでは「international students」をターゲットにしているのに対応していない矛盾

**改善案:**
- WhisperKitの日本語モデル（`openai_whisper-large-v3`）を追加
- 録音開始前に言語選択UI（最低限: 英語・日本語・中国語・韓国語・スペイン語）
- 自動言語検出モードも用意（精度は下がるが便利）

**優先度: 最高** — これだけで利用者層が10倍に広がる

### 1-3. 録音画面がシンプルすぎて不安になる

**現状の問題:**
- RecordViewは大きなタイマーとボタンだけ
- 「ちゃんと録音できてる？」という不安をユーザーに与える
- 他の録音アプリと差別化できていない

**改善案:**
```
┌─────────────────────────────┐
│                             │
│    Intro to Psychology      │  ← 講義タイトル（自動 or 手入力）
│    Prof. Smith  |  Room 302 │  ← メタデータ（任意）
│                             │
│         01:23:45            │  ← タイマー
│                             │
│    ▁▂▃▅▇▅▃▂▁▂▃▅▇▅▃        │  ← リアルタイム音声波形
│                             │
│      ● Recording...         │  ← ステータス表示
│                             │
│    [ ⏸ ]      [ ■ Stop ]   │
│                             │
│    🔴 Live  |  44.1kHz     │  ← 品質インジケーター
└─────────────────────────────┘
```

**追加すべき要素:**
- リアルタイム音声波形（AVAudioRecorderの `averagePower` を利用）
- 録音前にタイトル入力フィールド（後から編集も可）
- 録音品質インジケーター（音が小さすぎる時の警告）
- 「録音中」の明確なビジュアル（パルスアニメーション等）

## 2. UI/UXの設計見直し

### 2-1. iOSアプリのタブ構成の再設計

**現状:**
```
[録音] [ライブラリ] [設定]
```

**問題点:**
- 録音は頻繁に使うが、設定はほぼ使わない
- ライブラリが講義の「ハブ」になるべきだが、タブの2番目で存在感が薄い

**改善案:**
```
[ホーム/ライブラリ] [録音] [設定]
```
または
```
[ホーム] [録音(中央大)] [プロフィール]
```

- ホーム画面 = ライブラリ + ダッシュボード統合
  - 最近の講義（3件）
  - 「録音を開始」ボタン（大きく）
  - 統計（今週の録音時間、総講義数）
  - 未同期の講義があれば通知バッジ
- 録音ボタンは中央に大きく配置（Instagram的なUI）
- 設定はプロフィール/アカウントに統合

### 2-2. ライブラリ画面の改善

**現状の問題:**
- フラットなリスト表示のみ
- 講義が増えると探しにくい
- 科目やタグでの整理ができない

**改善案:**
```
┌─────────────────────────────┐
│  My Lectures          [+]   │
│                             │
│  ┌─────┐ ┌─────┐ ┌─────┐  │
│  │ All │ │ CS  │ │ Bio │  │  ← フォルダ/タグ
│  └─────┘ └─────┘ └─────┘  │
│                             │
│  Today                      │
│  ┌───────────────────────┐  │
│  │ 📝 Intro to CS        │  │
│  │ 1h 23m  |  Completed  │  │
│  │ "Today we covered..." │  │
│  └───────────────────────┘  │
│                             │
│  Yesterday                  │
│  ┌───────────────────────┐  │
│  │ 🔬 Biology 101        │  │
│  │ 45m  |  Completed     │  │
│  └───────────────────────┘  │
│                             │
│  This Week                  │
│  ...                        │
└─────────────────────────────┘
```

**追加機能:**
- 日付別グルーピング（Today / Yesterday / This Week / Earlier）
- フォルダまたはタグ機能（科目ごとに整理）
- ソート切替（日付/長さ/タイトル）
- リスト表示/グリッド表示の切替

### 2-3. 講義詳細画面の改善

**現状の問題:**
- テキストがベタ貼りで読みにくい
- 長い講義の文字起こしはスクロールが大変
- 音声とテキストが連動していない

**改善案（最大のUX改善ポイント）:**

```
┌─────────────────────────────┐
│  ← Intro to Psychology      │
│                             │
│  ┌───────────────────────┐  │
│  │ ▶   advancement slider │  │  ← ミニプレイヤー（常時表示）
│  │ 12:34 ─────●── 1:23:45│  │
│  └───────────────────────┘  │
│                             │
│  [Transcript] [Summary] [Q&A]│ ← タブ切替
│                             │
│  ... So the key concept     │
│  here is that cognitive     │  ← 再生位置に連動してハイライト
│  behavioral therapy works   │
│  by changing thought        │
│  patterns...                │
│                             │
│  [Copy] [Share] [Save Web]  │
└─────────────────────────────┘
```

**核心的な改善:**
- **音声とテキストの同期表示**: `TranscriptionSegment` のタイムスタンプを活用し、再生中のセグメントをハイライト。テキストをタップすると該当箇所から再生
- **タブ切替**: Transcript / AI Summary / Exam Q&A をタブで切り替え
- **ミニプレイヤー**: 画面上部に常時表示、スクロールしても消えない
- **共有機能**: テキスト共有、PDF書き出し

### 2-4. Web版ダッシュボードの改善

**現状の問題:**
- 講義リストがシンプルすぎる
- Pro機能の価値が伝わりにくい
- モバイルWebでの使い勝手が悪い

**改善案:**
- ダッシュボードにスタディ統計を追加（今週の学習時間グラフ）
- AI Summary/Exam Modeの結果プレビューをカード表示
- Pro機能の「お試し」を月3回無料にする（フリーミアムの鉄則）
- レスポンシブ改善（モバイルWebでの閲覧体験）

## 3. ユーザーが「手放せなくなる」ための機能追加

### 3-1. スタディストリーク（習慣化メカニズム）

**概要:** Duolingo的な連続学習記録
```
🔥 5 Day Streak!
Record or review a lecture to keep it going
```

- 毎日録音 or 復習するとストリーク継続
- 週間/月間レポート（「今週は12時間分の講義を記録しました」）
- Push通知:「今日の講義を録音しませんか？」

**なぜ重要:**
- 習慣化 = リテンション率の劇的改善
- 実装コストが低い（ローカルカウンター + 通知）

### 3-2. クイックアクション（録音開始の高速化）

**概要:** アプリを開いてから録音開始まで0秒にする

- **ウィジェット**: ホーム画面に録音開始ウィジェット
- **ショートカット**: Siri「Hey Siri, 講義を録音して」
- **3D Touch**: アプリアイコン長押し → 「録音開始」
- **Apple Watch**: 手首から録音開始/停止（将来的に）

### 3-3. スマート検索

**現状:** タイトルとテキストの単純な部分一致検索

**改善案:**
- 「先週の心理学の講義で出てきたCBTの定義」のような自然言語検索
- 講義横断検索（全講義をまたいで検索）
- 検索結果から該当箇所にジャンプ（タイムスタンプ付き）

### 3-4. 講義の共有・エクスポート

**現状:** コピーボタンのみ

**必要な機能:**
- PDFエクスポート（タイトル・日付・文字起こし全文）
- テキストファイル書き出し
- 友達への共有リンク（Web版で閲覧可能）
- AirDrop共有

## 4. 競合分析に基づくポジショニング

### 4-1. 主要競合との差別化

| 機能 | Lecsy | Otter.ai | Notta | AudioPen |
|------|-------|----------|-------|----------|
| オフライン文字起こし | ✅ | ❌ | ❌ | ❌ |
| プライバシー（ローカル処理）| ✅ | ❌ | ❌ | ❌ |
| 無料枠 | 無制限録音 | 月300分 | 月120分 | 月3回 |
| 価格 | $2.99/月 | $16.99/月 | $13.99/月 | $18.99/月 |
| 多言語 | ❌(英語のみ) | ✅ | ✅ | ✅ |
| AI要約 | Pro | Pro | Pro | 全プラン |
| リアルタイム | ❌ | ✅ | ✅ | ❌ |

### 4-2. Lecsyの勝てるポジション

**「プライバシーファーストの学生向け講義録音アプリ」**

勝ち筋:
1. **価格**: $2.99は競合の1/5以下 — 学生の財布に優しい
2. **プライバシー**: 音声データがサーバーに送信されない唯一のアプリ
3. **オフライン**: Wi-Fiがない教室でも使える
4. **無制限録音**: 無料プランで録音制限なし

負けている点（早急に対処すべき）:
1. **多言語対応**: 英語のみは致命的 → 最優先で対応
2. **リアルタイム文字起こし**: 録音中にテキストが見えない → 中期目標
3. **コラボ機能**: 友達と講義ノートを共有できない → 将来的に

## 6. テクニカルな改善

### 6-1. パフォーマンス改善

| 問題 | 現状 | 改善 |
|------|------|------|
| ライブラリ検索 | NSAttributedStringをView内で毎回生成 | 検索結果をキャッシュ、debounce追加 |
| Live Activity | 毎秒更新 | 5秒間隔に変更（バッテリー節約）|
| AuthService | 1599行の巨大ファイル | 3ファイルに分割（Apple/Google/Session） |
| JSONストレージ | lectures.json に全データ | SwiftData or Core Data移行（100件超で遅くなる）|

### 6-2. エラーハンドリングの改善

**現状:** 技術的なエラーメッセージがそのまま表示される

**改善案:**
- ユーザーフレンドリーなエラーメッセージに統一
- ネットワークエラー → 「インターネット接続を確認してください」
- 認証エラー → 「もう一度サインインしてください」
- 文字起こしエラー → 「もう一度お試しください。問題が続く場合はアプリを再起動してください」
- エラー発生時のリカバリー導線を明確に

### 6-3. データの堅牢性

**現状の問題:**
- `lectures.json` にすべて保存 — ファイル破損で全データ消失
- バックアップなし
- iCloud同期なし

**改善案:**
- SwiftData/Core Dataへの移行（トランザクション安全性）
- iCloud同期（有料プランの付加価値にも）
- 定期的なローカルバックアップ

## まとめ

Lecsyは「オフライン×プライバシー×低価格」という明確な強みを持っています。
しかし現状は、その強みがユーザーに伝わる前に離脱されている可能性が高いです。

**最も重要な3つの改善:**

1. **初回体験を劇的に改善する** — オンボーディング + モデル事前DL
2. **多言語対応する** — 英語のみは機会損失が大きすぎる
3. **音声とテキストの同期再生** — これが「おっ」と思わせる体験になる

この3つだけで、「便利なメモアプリ」から「なくてはならない学習ツール」に変わります。
